# 자동화 운영안

## 총괄 PM 오케스트레이션

- 루트 총괄 PM은 `ops/PM_ORCHESTRATION.md`의 A~G 사이클로 상태 확인, 작업 선정, 배정, 결과 회수, 독립 검수, 수정 요청, 완료 처리를 수행한다.
- 프로젝트 에이전트 설정은 `.codex/agents/**`, 동시성·깊이 제한은 `.codex/config.toml`이 정본이다.
- 한 사이클의 전문 작업은 최대 3개다. 쓰기 작업은 서로 분리된 Worktree/브랜치가 확인된 경우에만 병렬 배정한다.
- 현재 도구가 custom agent 이름 선택을 노출하지 않으면 해당 TOML의 역할·통제를 일반 서브에이전트 요청문에 포함하는 방식으로 대체한다.
- 사용자가 별도 대화에 요청문을 수동 전달하는 방식은 서브에이전트를 사용할 수 없을 때만 대체 수단으로 사용한다.
- QA-005 읽기 전용 시험에서 서브에이전트 생성, 범위 제한 전달, 구조화된 결과 회수, 독립 검수와 `DONE` 처리를 확인했다. 상세 결과는 `ops/reports/QA-005-subagent-orchestration-test.md`에 있다.

## 현재 자동 수행 가능

| 주기 | 업무 | 현재 실제 명령·입력 | 결과 | 외부 반영 |
| --- | --- | --- | --- | --- |
| 매 변경 | 소스·보안·핵심 흐름 검사 | `pnpm test` | CI/내부 보고 | 없음 |
| 매 변경 | 배포 묶음 생성·검사 | `pnpm build`, `pnpm test:dist` | `dist/`·내부 보고 | 운영 배포 금지 |
| 주요 UI 변경 | 브라우저 스모크 | 현재 `pnpm test:browser`; CHG-A 정리 필요 | 내부 보고 | 없음 |
| 일간 | 보드·활성 작업·충돌·승인 대기 점검 | Git 상태와 `ops/**` | `ops/reports/daily/**` | 없음 |
| 주간 | 정적 데이터 품질 집계 | 승인 projection과 source registry 읽기 | `ops/reports/**` | 후보·원천 자동 공개 금지 |
| 주간 | SEO·콘텐츠 공백 점검 | 공개 HTML·sitemap·기존 콘텐츠 읽기 | `ops/reports/**` | 외부 게시 금지 |

서브에이전트의 결과는 총괄 PM 또는 `reviewer`가 검수해 `PASS`한 경우에만 `DONE`으로 전환한다. `REVISION_REQUIRED`는 같은 카드 범위에서 최대 2회만 재요청한다.

## 자동 수행 금지 또는 선행 조건 필요

- 운영 Supabase E2E: 스테이징 접근·테스트 계정 필요
- 이메일/SMS 발송: 채널·비용·수탁 정책 승인 필요
- NAVER 블로그·플레이스·지역검색 수집·검증·보강·추천·공개: ADR-016에 따라 실행 금지
- 공공데이터 최초 수집·변경 감지: D-23 승인 dataset의 이용허락·출처·갱신 조건을 BE-006 계약과 별도 실행 카드로 강제한 뒤 가능
- 사업자 상태 자동 재확인: D-24 사용자 결정, 사업자번호 요청 중 메모리 사용·즉시 폐기, 로그·DB·queue·backup 비저장과 server-side adapter E2E 승인 필요
- 자동 중복·주소·폐업·행사 분류는 검수 후보만 만들 수 있으며 자동 병합·폐업 확정·행사 가능 공개 금지
- 필드 만료 알림·오래된 가격 숨김·문의 임시 보류는 BIZ-003·OPS-008의 주기·예외·rollback 승인 후 가능
- 실제 배포·외부 게시·업체 연락·광고: 사용자 승인 필요
- 공통 파일 자동 수정: 총괄 PM의 작업별 단일 소유 배정 필요

자동화는 실패를 숨기지 않고 검토 가능한 보고서를 남긴다.
