# MKT-011 전달문

## 작업 목적

서울 돌잔치 업체 제한 파일럿을 실제로 시작하기 전에 대상 기준, 연락 문구, 횟수, 수신거부, 운영 용량, 성공·중단 기준을 D-26 승인안으로 정리한다. 실제 후보 추출·연락처 수집·업체 연락은 하지 않는다.

- 가격·수수료·Premium 혜택·SLA 확정
- 제품 코드, DB, API, 환경변수, 패키지·잠금 파일, CHG-A~C

## 구현 범위

- 서울 돌잔치 1차 대상군·제외 기준과 실제 후보 추출 전 선행 조건
- 무료 기본 등록·정보 수정의 가치 제안과 과장 없는 채널별 초안 문구
- 연락 빈도·후속 연락·중단·수신거부·중복 방지
- 주별 접촉 상한, 관리자 검수 용량, 적체 중단선
- 등록→관계 확인→필드 제출→검수→공개→문의 준비 지표
- 무응답·거절·분쟁·권리 미확인·제품 E2E 미완료 처리
- 12주 제한 파일럿안, 추천안·대안·선택지별 영향

## 범위 밖 문제 처리

실제 업체나 연락처를 찾거나 접촉하지 않는다. 제품·개인정보·가격·법률·비용 문제가 발견되면 직접 해결하지 말고 기존 백로그·승인과 중복을 확인해 결과 보고서의 `차단 조건/변경 요청`으로 총괄 PM에게 반환한다.

## 완료 조건

- 사용자가 대상·문구·채널·횟수·용량·성공/중단 기준을 항목별로 승인할 수 있다.
- 수신거부와 중복 연락 방지 절차가 있다.
- 제품 E2E와 D-23~D-25가 준비되지 않으면 연락을 시작하지 않는 게이트가 있다.
- 가격·수수료·예약·결제·성과를 미리 약속하지 않는다.
- 실제 외부 실행 흔적이 없다.

## 테스트 방법

- MKT-009·OPS-008의 대상·운영 큐·처리량·신뢰 문구를 교차 검사한다.
- 무응답, 거절, 수신거부, 중복, 검수 적체, 분쟁, 문의 경로 실패 시나리오를 정적 검사한다.
- 이메일·전화번호 등 실제 연락처와 비용·외부 게시가 생성되지 않았음을 확인한다.
