# FE-006 전달문 — 친근한 가족 커뮤니티형 홈페이지 개편

> 상태: `SUPERSEDED`. 현재 전략에 맞춘 초기 안전 홈 구현은 `ops/handoffs/FE-009.md`를 사용하며 이 전달문으로 제품 작업을 시작하지 않는다.

현재 상태는 `APPROVAL_REQUIRED`다. D-14 승인 전에는 제품 파일을 수정하지 않는다.

## 작업 목적

현재 12개로 분산된 홈 섹션을 6개 핵심 영역과 프리푸터로 정리하고, 원티드형 탐색성·잡플래닛형 비교 정보·닥터나우형 행동 명확성을 손품해방의 따뜻한 가족 커뮤니티 분위기로 구현한다.

## 구현 범위

- 디자인 명세의 6개 섹션과 프리푸터를 기존 데이터·링크로 구현한다.
- 홈의 반복 섹션을 통합하되 기존 URL과 쿼리 계약을 유지한다.
- 기존 계산기·체크리스트·가이드·업체 입점 링크를 유지하고, 기존 `community.html` URL을 사용하는 새 홈 진입 링크를 추가한다.
- `#how-it-works`와 `.process-list`는 소비자 최종 CTA의 조건 선택→업체 비교→견적 문의 흐름으로 통합하고, `#provider-join`은 별도 업체용 프리푸터로 유지한다.
- 3개 지정 뷰포트에서 반응형과 키보드 접근성을 검증한다.

## 범위 밖 문제 처리

업체 상세·목록·비교·입점 화면, 데이터 성능, 공통 토큰, 라우팅 문제를 발견해도 수정하지 않는다. 재현 경로와 영향을 총괄 PM에게 별도 후보로 보고한다.
