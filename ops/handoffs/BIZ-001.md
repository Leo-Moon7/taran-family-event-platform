# BIZ-001 전달문

당신은 손품해방 프로젝트의 `business-product` 전문 에이전트다. 혼자 작업하는 것이 아니므로 다른 변경을 되돌리거나 범위를 넓히지 않는다.

## 작업 목적

사용자가 승인한 D-15~D-19를 바탕으로 초기 정보 플랫폼의 출처 라벨, 공개 기준, 고객 수정 제안, 업체 권한·입점, 변경 이력과 분쟁 처리 원칙을 하나의 구현 가능한 정책으로 정리한다.

- CHG-A~C와 완료 보고서

## 구현 범위

- 외부 검색 파생 정보·이용자 제보·업체 제공·운영 확인의 라벨과 공개 가능 필드
- 검수 전/후 상태, 수정·반려·이의·삭제 요청·되돌리기·변경 이력
- 고객 수정 제안과 업체 권한·신규 입점의 인증·증빙·결과 안내 원칙
- C안 13개 행동의 현재 공개 가능 문구와 선행 구현 항목
- BE-004 데이터 계약과 OPS-008 운영 절차가 이어받을 입력·출력
- 미확정 사항과 별도 허락·계약이 필요한 사항의 명시적 분리

가격·수수료, 법률 자문, API 계약 체결, 제품·DB 구현, 외부 연락·게시를 수행하지 않는다. 범위 밖 문제를 발견하면 직접 수정하지 말고 기존 백로그와 중복 여부를 확인해 보고서의 `후속 후보`에 기록한다.

## 완료 조건

- D-15~D-19를 빠짐없이 반영한다.
- 출처별 허용 필드·라벨·검수·공개·수정·회수 절차가 서로 모순되지 않는다.
- 가상 활동, 평점·추천·인기, 검수 완료로 오인되는 표현을 금지한다.
- C안 행동별 현재/후속 문구와 BE-004·OPS-008 인계 항목이 명확하다.
- 사실, 사용자 승인, 에이전트 제안을 구분한다.

## 테스트 방법

- 세 감사 보고서의 수치·차단 조건과 정책 매트릭스를 줄 단위로 교차 대조한다.
- D-15~D-19 누락, 금지 표현, 모순, 범위 밖 약속을 정적으로 검사한다.
- 제품·외부 시스템·운영 데이터는 실행하거나 변경하지 않는다.
