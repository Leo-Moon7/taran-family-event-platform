# OPS-007 전달문 — vendor-dashboard 강제 리디렉션 핫픽스

## 작업 ID

OPS-007

## 작업 목적

Netlify의 정적 파일 shadowing으로 무시되는 `/vendor-dashboard.html` 301 규칙을 최소 수정해 `/admin/providers.html`로 일관되게 이동시키고 레거시 JavaScript 오류 노출을 차단한다.

현재 상태는 `APPROVAL_REQUIRED`다. 총괄 PM이 D-13 승인 완료를 통지하기 전에는 수정·커밋·배포를 시작하지 않는다.

## 먼저 읽을 문서

- `AGENTS.md`
- `ops/TASK_SPECS.md`의 OPS-007
- `ops/reports/QA-006-netlify-public-smoke.md`
- `ops/CHANGE_REQUESTS.md`의 CR-004
- `ops/FILE_OWNERSHIP.md`
- `ops/DEPENDENCIES.md`

## 수정 허용 경로

- `netlify.toml`

## 수정 금지 경로

- `_redirects`
- `vendor-dashboard.html`
- `vendor-dashboard.js`
- `admin/**`
- 그 외 모든 HTML·JS·CSS
- API, DB, 마이그레이션, 환경변수, 패키지·잠금 파일
- CHG-A~C

## 구현 범위

- `/vendor-dashboard.html` → `/admin/providers.html` 301 블록에 `force = true`를 추가한다.
- 다른 redirect·header 블록의 순서나 값을 변경하지 않는다.
- 빌드 및 배포 산출물 검사를 실행한다.
- 사용자 승인된 테스트 재배포 후 무추적 HTTP 응답과 세 뷰포트 브라우저 이동을 확인한다.

## 범위 밖 문제 처리

다른 리디렉션, 레거시 코드, 관리자 인증, Netlify 환경변수 문제를 발견해도 직접 수정하지 않는다. 파일·줄·재현 명령·영향을 총괄 PM에게 변경 요청으로 보고한다.

## 완료 조건

- `/vendor-dashboard.html`이 301을 반환한다.
- `Location`이 `/admin/providers.html`이다.
- 브라우저 최종 URL에서 관리자 로그인 안내가 표시된다.
- 세 뷰포트에서 가로 넘침이 없다.
- 새 탐색에서 `vendor-dashboard.js` 오류가 발생하지 않는다.
- 허용 파일 외 변경이 없다.

## 테스트 방법

- `pnpm build`
- `pnpm test:dist`
- `curl` 무추적 응답 확인
- 390×844, 768×1024, 1440×1000 브라우저 읽기 전용 검수
- 실제 로그인, 폼 제출, DB 쓰기는 수행하지 않는다.
