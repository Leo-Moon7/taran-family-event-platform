## 작업 정보

- 작업 ID: FE-012
- 작업명: 행사 분류 5종 통합과 실용형 체크리스트 개편
- 담당: frontend-design
- 상태: IN_PROGRESS

## 작업 목적

공개 화면의 행사 분류를 다음 5개로 단순화하고, 체크리스트는 행사 종류 하나만 고르면 준비 단계별 상세 목록이 바뀌는 형태로 개편한다.

1. 돌잔치·백일 (`kids`)
2. 환갑·칠순·팔순 (`parents`)
3. 결혼 준비 (`meeting`, 기존 `smallWedding` 포함)
4. 기념일·생신 (`anniversary`)
5. 기타 가족행사 (`other`, 기존 `familyGathering`·`memorial` 포함)

## 구현 범위

- 사용자에게 보이는 행사 선택을 5개로 통일한다.
- 새 ID를 만들지 않고 기존 대표 ID `meeting`, `other`를 사용한다.
- `smallWedding`은 `meeting`, `familyGathering`·`memorial`·`home`은 `other`로 자동 연결한다.
- 검색 문맥을 읽고 저장할 때 이전 행사 ID를 대표 ID로 정규화해 기존 북마크가 5개 선택 화면에서도 정확히 열린다.
- 이전 URL, 로컬 저장 체크리스트, 메모, 사용자 추가 항목을 삭제하지 않는다.
- 체크리스트 왼쪽에는 행사 종류, 진행률, 초기화만 둔다.
- 선택 사항인 행사 날짜는 본문 상단에 작게 둔다.
- 행사별 목록은 준비 단계로 나누고 각 할 일에 실용적인 설명과 권장 시점, 관련 기능 연결을 제공한다.
- 결혼 준비의 상견례·소규모 예식 항목과 기타 가족행사의 가족모임·추모 항목은 선택 항목으로 표시해 불필요한 일을 강요하지 않는다.

## 범위 밖 문제 처리

범위 밖 문제는 직접 수정하지 않는다. 파일·API·DB·라우팅·환경변수·패키지·공통 설정 변경이 필요하면 즉시 중단하고 총괄 PM에게 변경 요청으로 보고한다. 기존 dirty diff를 되돌리거나 정리하지 않는다.

## 완료 조건

- 5개 공개 분류가 홈, 계산기, 체크리스트와 지정 폼에서 일관된다.
- 이전 `smallWedding`, `familyGathering`, `memorial` 입력이 대표 분류로 정상 연결된다.
- 체크리스트 시작에 필요한 왼쪽 선택은 행사 종류 하나뿐이다.
- 5개 행사 모두 단계별 상세 목록과 실제 확인 포인트가 있다.
- 기존 완료 상태·메모·사용자 추가 항목이 보존된다.
- 390px, 768px, 1440px에서 가로 넘침과 콘솔 오류가 없다.
