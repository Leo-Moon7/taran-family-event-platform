# QA-013 전달문

## 작업 목적

업체 등록·소유권·사업자 상태·사진·후기 검수에서 필요한 개인정보와 증빙을 최소화하고, D-24 사용자가 수집 전 승인할 수 있는 보안·보유·접근·파기 결정안을 작성한다. 실제 개인정보나 증빙은 수집하지 않는다.

- 위 결과 파일 외 모든 파일
- 개인정보·사업자번호·관계 서류·신분증·사진/후기 증빙의 실제 수집·전송·저장

## 구현 범위

- 목적별 필수·선택·금지 항목과 더 적게 수집하는 대체 확인 수단
- 사업자 상태, 담당자 관계, 소유권, 편집 권한을 별도로 확인하는 기준
- 보유기간 선택안과 만료·철회·파기·법적 보존 충돌 처리
- 역할별 접근, 마스킹, 암호화, signed access, 감사 로그, 다운로드·복제 통제
- 공개 projection에 절대 포함하지 않을 항목
- 무권한 열람·로그 노출·과수집·삭제·분쟁·사고 대응
- 15개 이상 위협·오남용 시나리오와 추천안·대안·영향

## 범위 밖 문제 처리

법률 문구를 확정하거나 DB/RLS/Storage 설계·변경을 하지 않는다. 필요 사항은 `법률 확인`, `후속 기술 계약`, `사용자 승인`으로 분리해 파일·위험·추천 통제를 총괄 PM에게 보고한다.

## 완료 조건

- 사용자가 D-24를 목적·항목·보유·접근·파기별로 승인할 수 있다.
- 수집 전 중단 조건과 수집 후 사고·철회 절차가 명확하다.
- 공개 가능한 정보와 비공개 증빙이 분리된다.
- OPS-008의 역할·큐·상태와 모순이 없다.
- 법률적 확정이나 실제 수집으로 오해할 표현이 없다.

## 테스트 방법

- OPS-008 Q6·Q7·Q10·Q11·Q13과 역할 매트릭스를 대조한다.
- 과수집, 목적 외 이용, 무권한 열람, URL·로그 노출, 만료, 철회, 삭제, 분쟁 시나리오를 정적 검사한다.
- 비밀키·운영 권한·실제 개인정보에 접근하지 않았음을 확인한다.
