# FE-007 전달문

당신은 손품해방 프로젝트의 `frontend-design` 전문 에이전트다. 혼자 작업하는 것이 아니므로 다른 변경을 되돌리거나 지정 범위를 확대하지 않는다.

## 작업 목적

초기 커뮤니티가 비어 보이지 않도록 운영팀이 먼저 올린 실용 질문을 자연스럽게 보여주되, 가짜 회원·가짜 작성 시각·가짜 댓글·저장·공감 수로 실제 이용자 활동을 위장하지 않는다.

## 구현 범위

- 기존 31개 preview를 소수의 고유한 `운영팀 시작 질문`으로 정리
- 목록과 상세에서 운영팀 작성 또는 시작 질문임을 자연스럽게 확인 가능
- 가상 회원명, `2분 전` 같은 상대 시각, 가상 댓글·저장·공감 수 제거
- 각 질문에 실제 행사 준비에 도움이 되는 고유 본문과 사용자가 답하기 쉬운 질문 제공
- Supabase `published` 글이 있으면 실제 글을 먼저 표시
- 실제 글이 0건이거나 Supabase 미구성일 때 시작 질문 표시
- 기존 글쓰기·댓글 로그인 및 검수 대기 흐름과 URL 유지

가짜 회원 프로필·활동 수 생성, 실제 DB 게시글·댓글 생성, DB·RLS 변경, 검색·정렬 구현, 공통 디자인 변경은 하지 않는다. 범위 밖 문제는 수정하지 말고 파일·라인·재현 방법과 후속 후보만 보고한다. 공통 계약 변경이 필요하면 즉시 중단한다.

- 실제 published 글 우선, 0건/미구성 시 시작 질문
- 목록·상세·글쓰기·댓글 흐름 유지
- 허용 경로 외 변경 0
