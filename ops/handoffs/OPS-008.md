## 작업 목적

업체 신규 등록과 기존 페이지 소유권 신청부터 사업자·담당자 확인, 직접 정보 입력, 사진 동의, 관리자 검수, 반려·회수·이의·최근성 재확인까지의 내부 운영 절차를 설계한다.

## 구현 범위

- 신규 등록과 기존 페이지 소유권의 분리 흐름
- 사업자 상태와 담당자 관계, access grant의 분리 확인
- 업체 표준 필드 제출과 관리자 검수·반려·보완·회수
- 사진 권리·동의·철회·만료
- 후보·중복·주소·폐업·행사 capability·소유권·필드·사진·후기·오류·분쟁 큐
- 감사 이력, 역할별 접근, 개인정보 최소화, 내부 SLA 제안과 에스컬레이션

범위 밖 문제는 직접 고치거나 외부에 연락하지 말고 기존 위험·백로그와 중복 확인 후 후보로 보고한다. 운영 SLA나 법적 문구를 확정 약속처럼 작성하지 않는다.

## 완료·테스트

- 완료 조건: 역할·입력·증빙·결정·상태·사유·기록·에스컬레이션, 개인정보 최소화, 10개 이상 운영 시나리오, 승인 게이트가 완전함
- 테스트: 무권한 편집, 사업자 active지만 담당자 미확인, 사진 철회, 주소 충돌, 폐업 신호, 반려 후 재제출, 권한 회수, 이의·rollback 시나리오 검사
