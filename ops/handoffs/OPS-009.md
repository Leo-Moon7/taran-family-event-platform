# OPS-009 전달문

당신은 손품해방 프로젝트의 `backend-data` 전문 에이전트다. 혼자 작업하는 것이 아니므로 다른 변경을 되돌리거나 승인 범위를 넓히지 않는다. D-22는 2026-07-22 승인됐고 총괄 PM이 아래 파일의 OPS-009 단일 소유권을 확정했다.

## 작업 목적

ADR-016에 따라 NAVER 검색 파생 데이터를 로컬 원본에서 삭제하지 않고 공개 파일·HTML 참조·제품 소비 경로에서 가역적으로 격리한다.

## 먼저 읽을 문서

- `AGENTS.md`
- `docs/99_의사결정기록.md` ADR-016
- `ops/reports/QA-010-naver-dependency-inventory.md`
- `ops/reports/BE-005-public-data-verification-architecture.md`
- `ops/APPROVALS.md` D-22·D-27
- `ops/CHANGE_REQUESTS.md` CR-005
- `ops/RISKS.md` R-32·R-35·R-36
- `ops/TASK_SPECS.md`의 OPS-009 카드

## 수정 허용 경로

- `scripts/build/prepare-dist.mjs`
- `index.html`, `venues.html`, `provider.html`, `compare.html`, `inquiry.html`, `claim.html`, `venue.html`, `admin/index.html`, `admin/providers.html`
- `data.js`: NAVER 파생 배열 3개 병합 블록만 제거
- `scripts/pages/home.js`: `reviewLifecycleVerifiedData` 소비 제거와 기존 빈 상태 유지에 필요한 최소 변경
- `scripts/pages/venues.js`: 전체 공개 재고가 0개일 때의 준비 상태 문구만 최소 변경. 기존 CHG-B 접근성·이미지 변경은 그대로 보존
- `dist/**`: 빌드 명령이 생성하는 산출물로만 허용하며 직접 편집하지 않는다.

루트의 `app.js`, `venues.js`, `venue.js`는 레거시 공개 소비 코드지만 이번에는 소스를 삭제하거나 고치지 않는다. `app.js`와 `venues.js`는 공개 빌드 제외 목록에 추가하고, `venue.js`는 파생 데이터 스크립트가 로드되지 않아 후보를 받지 못하는 상태를 브라우저에서 검증한다.

## 수정 금지 경로

- PM이 D-22에서 지정하지 않은 모든 소스 파일
- `backend/data`, 로컬 DB, 수집기, 원본 데이터 삭제·재가공, 공공데이터 수집, 운영 DB·마이그레이션·환경변수·패키지·잠금 파일
- CHG-A, CHG-C와 완료 보고서
- CHG-B 중 `provider.html`은 위 4개 파생 `<script>` 태그 제거만, `scripts/pages/venues.js`는 위 0건 안내 분기만 허용한다. `scripts/pages/provider.js`, `styles/components/filter.css`는 수정하지 않는다.
- 루트 7개 `review-*.js`, `backend/data`, 로컬 DB, 수집기와 원본은 이동·이름 변경·삭제·내용 수정하지 않는다.

## 구현 범위

- 다음 7개 파일을 공개 번들과 source 공개 참조에서 제외한다. 소스 원본은 그대로 보존한다.
  - `review-candidates.js`
  - `review-coverage.js`
  - `review-venue-candidates.js`
  - `review-provider-candidates.js`
  - `review-lifecycle-candidates.js`
  - `review-lifecycle-verified.js`
  - `review-local-api-partners.js`
- `index.html`, `venues.html`, `provider.html`, `compare.html`, `inquiry.html`, `claim.html`, `admin/index.html`, `admin/providers.html`, `venue.html`의 파생 script 참조를 제거한다.
- `data.js`와 `scripts/pages/home.js`가 NAVER 후보를 실제 업체로 병합·표시하지 않게 한다.
- 공개 빌드에서 미사용 레거시 소비기 `app.js`, `venues.js`를 함께 제외한다. 소스는 보존한다.
- 홈·목록·상세·비교·문의·소유권·관리자에서 정직한 빈/준비 상태와 기존 URL을 유지한다.
- 원본 데이터·수집기·로컬 DB는 변경하지 않는다.

로컬 데이터 삭제, 신규 수집, 재가공, 외부 API 호출, DB 변경, 업체 화면 재설계, 배포 자동 실행은 하지 않는다. 총괄 PM이 승인하지 않은 공통 로더 변경이 필요해지면 범위를 넓히지 말고 `BLOCKED`와 변경 요청을 반환한다.
