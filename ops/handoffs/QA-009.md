# QA-009 전달문

당신은 손품해방 프로젝트의 `quality-security` 전문 에이전트다. 읽기 전용으로 NAVER 업체·후기 정보 이용 기준을 다시 검토한다. 법률 자문을 확정하거나 제품을 수정하지 않는다.

## 작업 목적

업체명·주소 같은 공개 사실을 이용하는 것과 NAVER 검색/API 결과를 대량 저장·가공·DB화하는 것을 구분해, 현재 프로젝트에서 실제로 가능한 범위를 비개발자도 이해할 수 있게 확정한다.

## 구현 범위

공식 NAVER/NCP 자료, 국가법령정보센터와 공공기관 공식 자료를 우선 사용해 다음을 각각 판정한다.

1. 업체명·업종·주소·공식 전화·공식 홈페이지 같은 기본 사실
2. NAVER 지역검색 결과를 화면에 그대로 표시
3. 지역검색 결과를 저장해 자체 업체 DB로 구성
4. 블로그 제목·요약·URL·작성일
5. 후기 내용을 요약·분류·점수화·추천에 사용
6. 대량 저장·보관·재배포
7. 출처 표시, 정정·삭제 요청, 보관 기간
8. 업체 공식 사이트·공공데이터·업체 직접 등록 등 대체 취득 경로

각 항목을 `가능`, `조건부 가능`, `별도 허락 필요`, `현재 방식 중지`로 나누고, 저작권과 별개로 API 계약·데이터베이스·개인정보·표시 문제가 생길 수 있음을 구분한다.

API 호출, 크롤링, NAVER 문의, 데이터 삭제·재수집, 제품 수정은 하지 않는다. 실제 콘솔 계약·별도 서면 허락을 확인하지 못하면 허용으로 추정하지 않는다.

## 완료 조건

- 공식 URL·확인일·조항 또는 공식 설명 근거
- 사실 정보와 표현물·검색 결과·DB화의 차이
- 현재 코드 흐름 항목별 재판정
- 안전하게 사용할 수 있는 최소 업체 데이터 수집·표시 방법
- D-15·D-18 권고
- 비개발자용 쉬운 결론과 법률 자문 한계
