# QA-002 전문 작업 요청문

작업 ID: QA-002

작업 목적은 현재 정적 업체 데이터 품질 수치를 재현 가능한 기준선으로 고정하는 것입니다. 제품 코드와 원천 데이터는 수정하지 마십시오.

## 먼저 읽을 문서

1. `AGENTS.md`
2. `ops/TASK_SPECS.md`의 QA-002 카드
3. `docs/00_프로젝트현황.md`
4. `docs/02_현재사이트분석.md`
5. `docs/05_업체데이터구조.md`
6. `docs/12_통합실행계획.md`

## 수정 범위

- 수정 허용 경로: `ops/reports/QA-002-data-quality-baseline.md` 신규 1개
- 수정 금지 경로: 위 파일 외 전부. 특히 `review-*.js`, `data.js`, `scripts/pages/**`, `scripts/core/**`, `admin-schema.sql`, `migrations/**`, `package.json`, `pnpm-lock.yaml`

## 구현 범위

- 정적 원천 파일과 `publicDirectoryData` 병합 결과를 읽기 전용으로 집계합니다.
- 전체·부분 합계, ID 중복, 지역, 행사 태그, 출처, 확인 상태, 필수 필드, 후기 근거, 이미지 확인 상태를 기록합니다.
- 실행 명령, 집계 정의, 결정적 표본 최소 10건을 남깁니다.
- 정적 데이터 결과와 운영 Supabase 미확인을 분리합니다.

공개 자격을 결정하거나 데이터를 수정하지 마십시오. 범위 밖 문제는 수정하지 말고 보고서의 `신규 작업 후보`에 재현 근거·영향·추천 담당 영역만 기록하십시오.

## 완료 조건과 테스트

- 같은 입력으로 수치를 다시 계산할 수 있어야 합니다.
- 부분합과 전체합, ID 중복 여부를 교차검산합니다.
- 기존 문서 수치와 다르면 차이 원인을 설명합니다.
- 임시 읽기 전용 명령은 가능하지만 저장소 의존성·테스트·산출물을 추가하지 마십시오.
