# MKT-008 전달문

당신은 손품해방 프로젝트의 `marketing-operations` 전문 에이전트다. 다른 에이전트가 커뮤니티 파일과 법률 보고서를 동시에 작업하므로 지정된 준비백과 파일만 소유하고 다른 변경을 되돌리지 않는다.

## 작업 목적

반복 초안인 준비백과 28개 중 초기 고객에게 가장 유용한 핵심 글을 실제 공개 가능한 수준으로 편집하고, 아직 미완성인 글은 목록·관련 글·직접 URL에서 숨긴다.

## 수정 금지 경로

- 그 외 모든 파일
- 특히 `articles.html`, `article.html`, 공통 CSS/JS, API, DB, 라우팅, 환경변수, 패키지, CHG-A~C

## 구현 범위

- 초기 핵심 글 최소 6개 선정·전면 편집
- 각 글에 고유한 제목·요약·본문·핵심 포인트·질문·실수·체크리스트·다음 행동 제공
- 확인 가능한 근거 링크와 검토일을 데이터에 기록하고 상세 화면에 표시
- 근거 없는 평균 금액·비율·기간, 법률 보장, 업계 관행 단정 제거
- 공개 글은 `published`, 나머지는 `draft`로 명시
- blog.js는 published 글만 목록·카테고리·관련 글·직접 상세에서 사용
- draft 또는 없는 slug를 다른 공개 글로 대체하지 않고 안내 상태 표시
- 기존 공개 URL 형식 보존

28개를 억지로 모두 공개하거나 같은 문단 구조로 재작성하지 않는다. 외부 이미지를 추가하지 않고, canonical·sitemap·DB 게시를 하지 않는다. 범위 밖 문제는 보고서에 후보로만 기록한다.

## 완료 조건

- 공개 글 6개 이상
- 공개 글 사이 반복 공통 문단 0
- 글별 근거 링크·검토일·고유 체크리스트
- 미완성 글 목록·관련 글·직접 URL 노출 0
- 존재하지 않는 slug가 다른 글로 잘못 열리지 않음
- 허용 경로 외 변경 0

## 테스트 방법

- 데이터 파싱과 상태별 개수 집계
- 문단 중복, 근거 없는 숫자, 확정 법률 표현 검사
- `pnpm test`, `pnpm build`, `pnpm test:dist`
- 공개 글, draft slug, 없는 slug의 목록·상세·관련 글 브라우저 스모크
