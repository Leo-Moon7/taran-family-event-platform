# QA-014 전달문

## 작업 목적

서울 돌잔치 업체 제한 파일럿 전에 공식 업무 연락처·발송 이력·수신거부 차단 데이터의 목적과 최소 범위를 D-29 승인안으로 정리한다. 수신거부 재접촉을 막으면서 연락정보를 불필요하게 수집·무기한 보유하지 않는 것이 목표다. 실제 업체나 연락처를 조사·수집·저장·발송하지 않는다.

## 구현 범위

- 공식 업무 연락처, 발송 시도·결과, 거절·수신거부 식별값 각각의 처리 목적
- 필수·선택·금지 필드와 개인 연락처 혼입 처리
- 원문·정규화·암호화·HMAC 등 식별 방식의 추천·대안·위험. hash/HMAC을 익명정보로 단정하지 않음
- 업체·지점·브랜드·대행사 단위의 수신거부 적용 범위
- 접근 역할, 열람·export·로그·분석·개인 도구 금지
- 보유기간·재검토·삭제·backup 복원·법적 보존 선택안
- 발송 직전 중복·수신거부 검사와 채널 변경·반송·오류 정정
- 20개 이상 위협·오남용 시나리오
- D29-1부터 이어지는 쉬운 사용자 승인표, 추천안·대안·선택 영향

## 범위 밖 문제 처리

법률 문구, 보유기간, HMAC 방식, DB/RLS/API를 확정하거나 구현하지 않는다. 공식 자료만 근거로 사실·추천·사용자 결정·법률 확인·후속 기술 계약을 분리한다. 범위 밖 문제는 직접 수정하지 않고 기존 백로그와 중복을 확인해 결과 보고서의 `변경 요청`으로 총괄 PM에게 반환한다.

## 완료 조건

- 사용자가 목적·필드·식별·범위·접근·보유·삭제를 항목별로 승인할 수 있다.
- D-24 공통 통제와 D-26 연락 정책, D-29 외부 영업 데이터 계약의 경계가 명확하다.
- 승인 전 연락처 수집·저장·발송 0과 수신거부 재접촉 0이 전 구간에서 일관된다.
- 공식 자료 URL·시행 상태·확인일과 법률 해석 한계가 기록된다.
- 실제 업체·연락처·외부 실행 흔적이 없다.
