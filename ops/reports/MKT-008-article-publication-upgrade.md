# MKT-008 — 준비백과 공개 품질 개선·노출 통제

- 작업 ID: `MKT-008`
- 기준일·콘텐츠 검토일: `2026-07-22`
- 판정: **PASS 권고**
- 수정 허용 범위: `blog-data.js`, `blog.js`, 이 보고서

## 1. 결과 요약

기존 28개 slug와 원본 초안은 삭제하지 않았다. 초기 핵심 글 6개를 주제별 고유 원고로 전면 편집해 `published`로 지정했고, 나머지 22개는 `draft`로 지정했다.

`blog.js`는 최종 상태가 `published`인 글만 준비백과 목록, 카테고리 수, 관련 글과 상세 본문에 사용한다. 초안 slug, 존재하지 않는 slug, slug가 없는 상세 주소는 다른 공개 글로 바꾸지 않고 `이 글은 아직 공개되지 않았습니다.` 안내를 표시한다.

- 기관명·URL·사용 범위를 담은 `sources`
- 고유한 본문, 핵심 요약, 질문, 체크리스트, 실수, 다음 행동

## 3. 근거 사용 원칙

### 확정 사실

- 공개 글 6개 모두 공식 정부·공공기관 자료 링크를 가진다.
- 계약 글은 예식업 기준이 돌잔치 등 모든 가족행사에 자동 적용된다고 쓰지 않았다.
- 예산 글은 시장 평균 금액, 권장 비율, 고정 예비비를 만들지 않았다.
- 성장영상 글은 출처 표기만으로 타인의 저작물을 자유 이용할 수 있다고 쓰지 않았다.
- 케이터링 글은 위생등급이나 업체 답변이 개별 음식의 안전을 보장한다고 쓰지 않았다.
- 후기 글은 할인·무료 제공·포인트 등 경제적 이해관계 표시를 안내한다.

### 편집 제안

- 비용 네 칸 분류, 일정 역산, 초대 문구와 체크리스트는 손품해방의 실용 편집안이다.
- 각 글은 법률·의료 자문이나 개별 업체의 안전·가격 보장이 아님을 `reviewScope`와 출처 설명에 표시한다.

### 수치·법률 표현 재검토

공개 본문에서 금액·비율·고정 기간·법률 보장 표현을 검색했다.

- `%`, `만원`, `D-숫자`, 고정 `일·개월·시간`: 없음
- `0원` 2건: 실제 가격 주장이 아니라 미확인 항목을 `0원`으로 입력하지 말라는 설명
- `자동 적용` 1건: 예식업 기준이 다른 가족행사에 자동 적용된다고 생각하지 말라는 부정 문맥
- 환급액이나 법률 결과를 보장하는 문장: 없음

## 4. 상태 필터 구현

### 준비백과 목록

- 원본 배열을 `allPosts`로 보존한다.
- `status === "published"`인 글만 `posts`에 포함한다.
- 대표 글, 목록, 글 수, 카테고리 수는 `posts`만 사용한다.
- 현재 목록 표시는 `6개`다.

### 상세와 관련 글

- slug 조회에는 전체 인벤토리를 사용해 초안과 없는 주소를 구분할 수 있게 한다.
- 조회 글이 없거나 `published`가 아니면 공개 본문과 관련 글을 렌더링하지 않는다.
- 관련 글 후보는 처음부터 공개 글 배열만 사용한다.
- 게시일, 검토일, 출처 링크와 자료별 사용 범위를 상세에 표시한다.
