# BIZ-004 공개 신뢰 라벨·최근성 D-25 결정 패킷

- 작업 ID: `BIZ-004`
- 작성 기준일: 2026-07-22 (Asia/Seoul)
- 문서 상태: **결정안 작성 완료 / 공개 명칭·문구·주기는 D-25 사용자 승인 전 미확정**
- 적용 범위: 공개 업체 정보의 신뢰 라벨, 필드별 확인일, 비교·견적 문의의 최근성 경계
- 비적용 범위: 제품 구현, DB/API/라우팅 변경, 실제 공공데이터 호출, 사업자번호·증빙 수집, 업체 연락, 가격·수수료 결정

## 1. 1분 결정표

아래 네 항목은 서로 따로 승인하거나 수정할 수 있다. **추천 일괄안은 `D25-A1 + D25-B1 + D25-C1 + D25-D1`**이다. 이 보고서의 추천은 아직 확정 정책이 아니며, 사용자가 승인한 뒤에만 후속 계약과 화면 문구의 기준이 된다.

| 결정 | 추천 선택 | 쉬운 뜻 | 다른 선택을 할 때 영향 |
| --- | --- | --- | --- |
| `D25-A` 업체 출처 라벨 | **A1 `업체 제공 정보`** | 관계가 확인된 업체 담당자가 제공한 정보라는 뜻 | A2 `업체 직접 제출`은 제출 경로가 더 분명하지만, 모든 정보가 담당자 본인이 화면에 직접 입력했다는 오해가 생길 수 있음 |
| `D25-B` 여섯 라벨 | **B1 2절의 쉬운 명칭·설명 승인** | 출처·검수·사업자·최근성을 하나의 배지로 합치지 않음 | B2 이름은 유지하고 보조 설명만 다시 쓰면 구현이 늦어지지만 공개 문구를 더 다듬을 수 있음 |
| `D25-C` 여러 라벨 표시 | **C1 경고 우선, 모바일 최대 2개** | 지금 주의할 상태를 먼저 보고, 세부 근거는 펼쳐 봄 | C2 출처 우선은 정보 출처는 빨리 보이나 오래되거나 확인 중인 상태가 뒤로 밀릴 수 있음 |
| `D25-D` 재확인 주기 | **D1 단계형 90/120/180/365일** | 가격은 빨리 숨기고, 일반 정보·문의·권한은 위험에 맞게 다르게 관리 | D2 엄격형은 오류 위험은 줄지만 표시·문의 가능 업체가 빨리 줄고 운영량이 늘며, D3 단순형은 이해는 쉽지만 오래된 가격 노출 위험이 커짐 |

사용자는 예를 들어 `D25-A1, B1, C1, D1 승인`처럼 답할 수 있다. 일부만 바꾸려면 해당 코드와 원하는 문구 또는 경계를 함께 지정하면 된다.

## 2. 추천 여섯 라벨 사전

여섯 라벨은 업체 전체의 등급이 아니다. **현재 화면에 보이는 특정 정보가 어디서 왔고, 무엇을 어디까지 확인했으며, 다시 확인할 때가 됐는지**를 설명한다.

| 추천 라벨 | 한 줄 설명 | 자세한 뜻 | 이 라벨이 뜻하지 않는 것 |
| --- | --- | --- | --- |
| `공공데이터 등록` | `공공데이터에 등록된 기본 정보를 바탕으로 표시했어요.` | D-23에서 별도 승인된 데이터셋의 허용 필드에 출처와 기준일이 있고, 공개에 필요한 검수를 통과한 경우에만 해당 필드에 표시 | 입점, 추천, 돌잔치 가능, 현재 영업 최종 확정, 현재 가격·인원·주차·사진 확인 |
| `정보 확인 중` | `새로 들어오거나 바뀐 정보를 확인하고 있어요.` | 신규 제출, 변경, 중복, 주소 충돌, 사업자 상태, 행사 가능, 분쟁 중 하나를 검토 중이며 새 값을 확정 비교 조건으로 사용하지 않는 상태 | 곧 승인된다는 약속, 사실 확정, 문의 가능 |
| `업체 제공 정보` | `관계가 확인된 업체 담당자가 제공한 정보예요.` | 승인된 업체 관계에서 제출된 필드에 제공일과 조건이 있고, 공개값은 관리자 검수를 통과한 경우에 표시하는 **출처 라벨** | 손품해방의 보증, 독립기관 확인, 사업자 정상, 미래에도 같은 가격·조건 보장 |
| `사업자 확인` | `업체가 제출한 사업자등록정보의 상태를 확인했어요.` | D-24에서 수집·조회가 허용된 경우에 한해, 제출된 사업자등록정보의 상태와 업체 연결을 정해진 시점에 확인한 상태 | 담당자의 재직·대리 권한, 페이지 소유권, 안전·품질, 가격·시설·사진 전체 확인 |
| `관리자 검수` | `손품해방 운영자가 이 항목의 출처와 표시 조건을 검수했어요.` | 운영자가 **해당 필드 또는 필드 묶음**의 출처, 단위, 포함·제외 조건, 권리, 결측 처리와 표시 적합성을 검수 | 공공기관 인증, 업체 전체 인증, 계약 이행 보증, 미래 가격·재고 보장 |
| `최근 정보 확인 필요` | `시간이 지났거나 변경 신호가 있어 다시 확인이 필요해요.` | 정해진 주기 초과, 업체가 적은 유효기간 만료, 출처·주소·사업자 충돌, 이의제기 또는 재확인 실패가 있는 필드에 표시 | 폐업 확정, 정보 오류 확정, 자동 삭제 |

### 사용자에게 보여 줄 보조 문구 추천

- 공공데이터 필드: `돌잔치 가능 여부와 가격을 확인한 뜻은 아닙니다.`
- 업체 제공 필드: `제공일과 적용 조건을 함께 확인해 주세요.`
- 사업자 상태: `담당자 권한이나 가격·시설 전체를 확인한 뜻은 아닙니다.`
- 관리자 검수 필드: `이 항목의 출처와 표시 조건을 검수했어요.`
- 오래된 필드: `마지막 확인일을 보고 문의 전에 다시 확인해 주세요.`
- 결측값: `정보 없음은 0원·불가·없음을 뜻하지 않아요.`

## 3. `업체 제공 정보`와 `업체 직접 제출` 결정

### 추천: `업체 제공 정보`

이 명칭을 추천하는 이유는 세 가지다.

1. D-17 승인 내용, `docs/99_의사결정기록.md`의 ADR-014·ADR-016 업체 참여 문장, `docs/04_사용자흐름.md`, BE-005, OPS-008이 이미 이 표현을 사용한다.
2. 이메일·서류·운영자 대리 입력처럼 향후 허용될 수 있는 제출 경로까지 포괄한다. `직접`이라는 단어 때문에 화면에 담당자가 손수 입력했다고 오해할 가능성이 더 낮다.
3. 이 라벨은 **제공 주체**만 설명한다. 별도의 `관리자 검수`, `사업자 확인`, 날짜 라벨과 자연스럽게 조합된다.

추천 전체 문구는 다음과 같다.

> **업체 제공 정보**  
> 관계가 확인된 업체 담당자가 제공한 정보예요. 제공일과 적용 조건을 함께 확인해 주세요.

### 대안: `업체 직접 제출`

장점은 공공데이터와 달리 업체 쪽에서 제출했다는 행동이 분명하다는 점이다. 반면 사용자는 `직접`을 `관리자 검수 없이 즉시 반영`, `업체 대표 본인이 입력`, `플랫폼이 내용까지 확인`으로 넓게 받아들일 수 있다. 이 대안을 선택하더라도 보조 문구에 `제출한 정보이며, 손품해방의 보증을 뜻하지 않아요`를 붙여야 한다.

### 용어 적용 규칙

- 공개 라벨은 D-25에서 선택한 **한 표현만** 사용한다. 두 표현을 같은 뜻의 별도 배지로 만들지 않는다.
- 내부 source 값 `provider_supplied`는 공개 명칭과 분리한다.
- `업체 제공 정보`는 출처, `관리자 검수`는 특정 필드의 공개 적합성, `사업자 확인`은 제출 사업자 상태다. 셋을 한 배지로 합치지 않는다.
- `업체 제공 정보`만으로 문의 버튼을 열지 않는다. 문의에는 T6 조건이 추가로 필요하다.

## 4. 라벨을 함께 표시하는 순서와 모바일 축약안

### 추천 순서: 지금 해야 할 행동을 먼저

한 필드에 여러 라벨이 적용되면 다음 순서로 표시한다.

1. 차단·주의: `정보 확인 중` → `최근 정보 확인 필요`
2. 출처: `업체 제공 정보` 또는 `공공데이터 등록`
3. 필드 검수: `관리자 검수`
4. 업체 단위 상태: `사업자 확인`
5. 바로 뒤 또는 펼침 영역: 제공일·기준일·확인일과 적용 조건

`정보 확인 중`과 `최근 정보 확인 필요`가 동시에 해당하면, 새 값이나 분쟁으로 현재 값을 쓰지 못하는 경우 `정보 확인 중`을 먼저 표시한다. 아직 쓸 수 있지만 오래된 기존값이면 `최근 정보 확인 필요`를 먼저 표시한다. 가격처럼 값이 숨겨진 경우에는 숨긴 이유와 마지막 확인일만 보여 준다.

### 화면별 규칙

| 위치 | 추천 표시 | 이유 |
| --- | --- | --- |
| 목록 카드 | 카드에서 실제로 보여 주는 요약값의 `주의 1개 + 출처 1개`만 표시 | 업체 전체가 같은 출처·날짜라는 오인 방지 |
| 상세 필드 | 주의 → 출처 → 관리자 검수 → 날짜·조건 순으로 전부 표시 | 해당 값의 근거와 범위를 재현 |
| 비교표 | 각 비교값 옆에 주의·출처·확인일을 표시하고, 공통 설명은 표 상단에 한 번 제공 | 업체 전체 배지로 가격·주차·시설을 뭉치지 않음 |
| 문의 영역 | T6 충족 여부와 문의 안내를 별도 표시 | 신뢰 라벨을 문의 가능 배지로 오인하지 않음 |

### 모바일 축약

- 한 줄에 최대 2개 칩만 노출한다. 첫 칩은 가장 보수적인 주의 상태, 둘째 칩은 그 값의 출처다.
- 주의 상태가 없으면 출처와 `관리자 검수`를 표시할 수 있다.
- 나머지는 `확인 정보 더보기 N개`로 펼친다. 짧지만 모호한 `검수됨`, `인증`, `공식` 같은 새 축약어를 만들지 않는다.
- 카드의 여러 요약 필드가 서로 다른 출처라면 업체 전체에 하나의 출처 배지를 붙이지 않고 `항목별 출처 보기`로 안내한다. 이는 일곱 번째 신뢰 라벨이 아니라 펼침 동작 문구다.
- 스크린리더에는 숨긴 라벨까지 순서대로 읽히게 하고, 색상만으로 주의 상태를 구분하지 않는다.

## 5. T5 공개·비교와 T6 견적 문의의 차이

| 구분 | T5 `field_reviewed` | T6 `inquiry_ready` |
| --- | --- | --- |
| 쉬운 뜻 | 공개하려는 **각 정보**의 출처와 조건을 검수한 단계 | 실제로 견적 문의를 받아 전달·열람·응답할 준비까지 확인한 단계 |
| 가능한 행동 | 다른 공개 gate를 모두 통과하면 기본 페이지 표시. 같은 비교군·단위·조건·최근성까지 맞으면 비교 가능 | T5·비교 자격에 더해 모든 문의 gate를 통과하면 `같은 조건으로 견적 문의` 가능 |
| 추가로 필요한 것 | 안정된 업체 ID, 중복 해소, 행사 가능 근거, 권리, 유효한 필드, 공개 검수 | 승인된 담당자 관계와 활성 권한, 현재 사업자 상태, 문의 수신 동의, 업무 연락·응답 담당자, 180일 이내 핵심 정보, 문의 E2E |
| 아직 뜻하지 않는 것 | 추천 업체, 문의 가능, 예약 가능 | 예약·결제·계약 완료, 실시간 재고, 가격 보장 |

T5는 공개·비교의 **필요조건이지 단독 충분조건이 아니다**. T6도 예약·결제 상태가 아니라 견적 문의를 받을 준비 상태다. `quote_only` 업체가 T5이면 가격 모델은 설명할 수 있지만 문의 버튼은 열지 않는다. 같은 업체가 T6까지 통과한 경우에만 같은 조건의 견적 문의를 열 수 있다.

## 6. 추천 최근성 주기 `D25-D1`

### 일수 계산과 우선순위

- 기준일 다음 날을 1일로 세는 KST 달력일 기준을 추천한다.
- 90일째는 현재 구간의 마지막 날, 91일째는 재확인 경고 시작, 120일째는 경고 구간의 마지막 날, 121일째는 가격 숨김 시작, 180일째는 문의 허용 검토의 마지막 날, 181일째는 문의 자동 중지 시작으로 한다.
- 담당자 관계·편집 권한은 365일째를 마지막 유효일, 366일째를 재확인 전 권한 중지 시작으로 제안한다.
- 업체가 적은 `valid_until`이 더 빠르거나 권리 철회, 폐·휴업 신호, 주소 충돌, 분쟁, 관리자 반려가 생기면 일수보다 그 사건을 먼저 적용한다.
- 단순 수집일, 행 `updated_at`, 관리자 형식 검수일은 값의 최근 확인일을 자동으로 갱신하지 않는다.

### 90/120/180/365일이 실제로 바꾸는 것

| 대상 | 0~90일 | 91~120일 | 121~180일 | 181~365일 | 366일 이상 |
| --- | --- | --- | --- | --- | --- |
| 가격·식대·대관료·취소·추가비 | 조건이 완전하면 금액 표시·비교 가능 | 금액은 보이되 `최근 정보 확인 필요`; 가격 정렬·최저 표현·확정 필터에서 제외 | 금액 숨김, `가격 정보 재확인 중`; 기본 페이지는 다른 gate가 유효하면 유지 | 금액 숨김 유지 | 동일 |

### 주기별 이용자·운영 영향

- **90일:** 변동성이 큰 가격과 정책을 현재값으로 취급하는 상한이다. 짧아서 신뢰에는 유리하지만 업체 재확인 업무가 늘어난다.
- **120일:** 가격을 바로 지우기 전 30일 경고 구간의 끝이다. 이용자는 마지막 금액을 참고할 수 있지만 가격순·최저 표현에서는 빠져 오인을 줄인다.
- **180일:** 일반 조건과 문의 준비의 최종 상한이다. 181일부터 오래된 시설값은 비교값을 대체하고 문의는 자동 중지하므로 문의 가능 업체 수가 줄 수 있다.
- **365일:** 담당자 관계·편집 권한의 정기 재확인 상한이다. 가격이나 사업자 상태를 1년 동안 현재로 본다는 뜻이 아니다. D-24의 개인정보·증빙 정책과 함께 운영해야 한다.

## 9. 금지 표현과 문맥 검사

아래 표현은 별도 근거·기능·승인이 없으면 공개 문구로 사용하지 않는다.

- `검증 완료`, `공식 인증`, `100% 검증`, `믿을 수 있는 업체`
- `추천 업체`, `인기`, `서울 1위`, `베스트`, `최저가`, `실시간 가격`
- `예약 가능`, `즉시 예약`, `예약 확정`, `결제 가능`
- `사업자 확인`을 가격·시설·사진·안전까지 확인했다는 뜻으로 넓히는 문구
- 결측값을 `무료`, `없음`, `불가`, 숫자 0으로 바꾸는 문구

신뢰 라벨은 광고·순위·추천이 아니며, 미확정 Premium 가격·리드 과금·광고 혜택·중개 수수료와 연결하지 않는다.

## 10. 사실·제안·사용자 결정 분리

### 이미 확정된 사실

- D-01~D-03: 후기 수는 공개 자격이 아니고, `api_collected`는 검증이 아니며, 결측값은 추정하지 않는다.
- D-04·ADR-011·ADR-016: 1차 전환 종점은 비교·견적 문의·업체 응답이며 예약·결제는 보류한다.
- D-17: 승인된 업체 담당자의 자사 정보는 초기 관리자 검수 후 `업체 제공 정보`와 확인일로 표시한다.
- ADR-016: 신뢰 상태를 출처·사업자/관계·필드 검수·최근성 축으로 나누고 하나의 배지로 합치지 않는다.
- T5는 공개·비교 자격 검토의 필요조건이고, T6만 견적 문의 자격을 검토한다.

### 이 보고서의 추천안

- 공개 단일 명칭 `업체 제공 정보`
- 2절의 여섯 쉬운 라벨 설명
- 경고 우선 순서와 모바일 최대 2개 표시
- D1 단계형 90/120/180/365일 경계와 KST 일수 계산

### 사용자가 D-25에서 결정할 것

1. `D25-A1` 또는 `A2`: 업체 출처 라벨의 최종 명칭
2. `D25-B1` 또는 `B2`: 여섯 라벨 이름과 보조 문구
3. `D25-C1` 또는 `C2`: 함께 표시할 순서와 모바일 축약
4. `D25-D1`, `D2`, `D3` 또는 수정안: 가격·일반 필드·문의·권한 재확인 경계

### D-25가 승인하지 않는 것

- 공공데이터 원천 채택·실제 호출(D-23)
- 사업자번호·담당자 관계·사진/후기 증빙의 수집·보유·접근·파기(D-24)
- 업체 외부 연락, 운영량, 서울 돌잔치 파일럿 시작(D-26)
- 운영 DB/API/Storage, 제품 코드, 배포, 최종 병합
- 가격·수수료·광고·유료 상품·예약·결제
