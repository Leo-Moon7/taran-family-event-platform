## 3. 실제 재현 수치와 집계 정의

QA-002와 동일하게 5개 파일을 공개 화면 로드 순서로 격리 실행한 뒤의 `window.publicDirectoryData`를 정적 병합 결과로 사용했다. 운영 Supabase override는 포함하지 않았다.

### 3.1 공개 배열과 서울

| 항목 | 실제 재현 수치 | 집계 기준 |
| --- | ---: | --- |
| 공개 정적 병합 결과 | 4,960 | `window.publicDirectoryData.length` |
| 원천 부분합 | 30 + 39 + 4,891 | 후기 후보 + 공개된 검증 라이프사이클 + 공개된 지역 API |
| `region=서울특별시` | 999 | 최종 공개 정적 병합 결과의 원문 `region` |
| 서울 `kids` 태그 | 889 | 서울 999건 중 `eventTags`에 `kids` 포함 |
| 서울 `subcategory=돌잔치·백일` | 120 | 서울 999건 중 정확한 문자열 일치 |
| 서울 외부 후기 근거 보유 | 6 | 서울 999건 중 `externalReviews.length > 0` |
| 서울 `officialVerification.status=verified` | 6 | 서울 999건 중 상태값 정확한 일치 |
| 원본 `reviewLifecycleVerifiedData`의 서울 | 7 | 공개 필터 전 원천 배열 소속만 집계 |

차이를 만드는 레코드는 `review-lifecycle-7a0532a107c4` 남산뜰 1건이다. 이 레코드는 서울이고 외부 후기 링크 4개가 있지만 `publicationStatus=hidden`, `sourceStatus=비공개 대기`, `officialVerification.status=unmatched`라 최종 공개 병합에서 제외된다.

과거의 서울 `1,000`과 외부 후기 `7`은 이 hidden 레코드를 공개 결과에 포함하면 재현된다. 그러나 과거 문구의 `verified 7`은 상태값 집계로는 재현되지 않는다. 원본 배열 이름이 `reviewLifecycleVerifiedData`라는 이유로 서울 원천 7건 전체를 verified로 간주한 것으로 추정되며, 실제 `officialVerification.status=verified`는 원본에서도 6건이다.

### 3.2 주소

| 항목 | 실제 재현 수치 | 해석 제한 |
| --- | ---: | --- |
| 최상위 `address` 또는 `roadAddress` | 0 | 최상위 표준화 필드가 없다는 뜻 |
| `officialVerification.roadAddress` | 4,850 | QA-002의 좁은 도로명 주소 정의 |
| `officialVerification.address` 또는 `.roadAddress` | 4,930 | 중첩 공식 확인 객체의 주소 값 존재 |
| `detailFacts`의 키에 `주소`가 포함된 값 | 4,930 | 표시용 보조 필드 포함 |
| 위 보조 주소를 합친 넓은 정의 | 4,930 | 중복 제거된 업체 수 |
| `officialVerification.roadAddress`는 없고 보조 주소만 있는 업체 | 80 | `4,930 - 4,850` |
| 넓은 정의에서도 주소가 없는 업체 | 30 | 후기 기반 후보 원천에 집중 |

`4,850`과 `4,930`은 서로 경쟁하는 수치가 아니라 정의가 다른 수치다. 어느 쪽도 주소가 최상위 표준 필드로 승격됐거나 주소 정확성이 검증됐다는 뜻은 아니다.

- 현재 작업 트리에는 Playwright 패키지 설정과 검사 파일이 있으나 CHG-A 검토 대기 상태다. 현 환경의 지정 Chromium 실행 파일은 없고 현행 브라우저 통과 여부는 미확인이다.
