## 핵심 방향

초기 홈은 입점 업체가 많거나 즉시 견적을 받을 수 있다고 가정하지 않는다. 방문자가 바로 업체 정보를 찾고, 원문 후기를 확인하고, 자신이 아는 내용을 보완할 수 있는 정보 작업 화면으로 구성한다.

## 공통 상단 메뉴

- 손품해방 로고
- 행사 계산기
- 체크리스트
- 준비백과
- 정보 나눔
- 업체 수정
- 업체 입점
- 로그인
- `내 준비 현황`

`업체 수정`은 기존 업체의 정보 수정 제안과 업체 담당자의 권한 요청을 안내하고, `업체 입점`은 신규 업체 등록으로 구분한다.

## 1. 바로 업체 찾기

- 질문: `어떤 가족행사를 준비하세요?`
- 행사 종류, 지역, 예상 인원 선택
- 주 행동: `조건에 맞는 업체 찾기`
- 보조 행동: `예산 계산하기`, `준비 체크리스트`
- 긴 서비스 소개와 큰 홍보 사진은 넣지 않는다.

## 2. 행사별 빠른 시작

- 대안 B의 사진 중심 이미지 아코디언 레이아웃을 유지한다.
- 돌잔치·백일, 칠순·팔순, 결혼·가족모임, 장례·추모, 기타 행사로 바로 진입한다.
- 업체 수·후기 수 등 검증되지 않은 숫자는 표시하지 않는다.

## 3. 후기 기반 업체 둘러보기

`후기 많은 업체`, `평점 좋은 업체`, `추천 업체`라는 표현은 초기에는 사용하지 않는다.

권장 제목: `후기에서 찾아본 업체`

업체 카드에 표시할 내용:

- 업체명, 지역, 행사 유형
- `공개 후기 원문 확인` 상태
- 가장 최근에 확인한 원문 날짜
- 후기에서 언급된 항목과 `확인 필요` 구분
- 정보 출처와 마지막 확인일

제공할 행동:

- `업체 정보 보기`
- `후기 원문 보기`
- `정보 수정 제안`

외부 후기 링크 수는 네이버 전체 후기 수가 아니므로 집계 정의가 승인되기 전에는 순위나 인기 지표로 사용하지 않는다. 평점은 공통 기준과 검증 방법이 없어 표시하지 않는다.

## 4. 준비 도구

콘텐츠를 읽기 전에 바로 사용할 수 있도록 준비백과보다 먼저 배치한다.

- 준비 체크리스트
- 예산 범위 계산기
- 주 행동: `체크리스트 열기`, `예산 계산하기`

현재 기능을 연결하고 실제 저장 여부와 로그인 필요 여부를 화면에서 명확히 안내한다.

## 5. 준비백과와 정보 나눔

- 행사별 준비 순서
- 놓치기 쉬운 항목
- 업체에 확인할 질문
- 먼저 준비한 사람의 경험
- 정보 나눔 게시판
- 행동: `준비백과 전체 보기`, `정보 나눔 보기`

최신 또는 대표 글만 노출하고 가상의 조회수·반응 수를 만들지 않는다.

## 6. 공동 편집 참여 홍보 화면

홍보 화면은 유지하되 입점 수나 즉시 견적을 홍보하지 않는다.

권장 문구:

> 알고 있는 정보 하나가 다음 가족의 손품을 줄입니다.

행동 버튼:

- 고객: `내 경험으로 정보 보완하기`
- 업체: `업체 수정 권한 요청`
- 신규 업체: 작은 링크 `새 업체 등록하기`

하단에는 `외부 후기`, `이용자 제보`, `업체 제공`, `운영 확인`이 서로 다른 정보라는 짧은 안내를 둔다.

## 공동 편집 표시 원칙

- 고객 제안은 바로 공개하지 않고 관리자 검수 후 반영한다.
- 고객 견적은 시점·인원·포함 조건과 함께 사례로만 표시한다.
- 업체 입력은 `업체 제공 정보`라고 표시한다.
- 운영 확인을 받지 않은 정보에는 `확인 필요`를 표시한다.
- 수정 전후 값, 출처, 제안자 유형, 승인일을 이력으로 남긴다.
