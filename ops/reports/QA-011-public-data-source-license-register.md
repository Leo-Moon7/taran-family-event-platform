## 1. 결론

현재 공식 메타데이터만으로 D-23의 우선 검토 대상으로 올릴 수 있는 원천은 행정안전부의 전국 인허가 조회서비스 3개다.

1. `15154916` 일반음식점
2. `15154897` 관광식당
3. `15155090` 관광숙박업

서울 열린데이터광장의 `OA-16094` 일반음식점과 `OA-16091` 관광식당은 공공누리 제1유형으로 권리 조건이 더 명확하지만, 위 전국 서비스와 같은 지방행정 인허가 계열을 서울 범위로 다시 제공하는 중복 후보이다. **같은 업종에서 전국형과 서울형을 동시에 독립 근거로 세지 말고 D-23에서 하나의 canonical lineage만 선택**해야 한다.

국세청 `15081808`은 업체가 자발적으로 제출한 사업자등록번호의 휴·폐업 상태를 확인하는 조건부 보조 원천이다. 후보 발견 수단이 아니며 기존 4,960건 역조회, 대표자명 기반 대량 진위조회, 사업자등록번호 공개에는 사용할 수 없다. D-24 업체 제출·동의 흐름과 개인정보·secret·보관 통제가 확정된 뒤에만 쓸 수 있다.

나머지는 현재 채택할 수 없다. 상가(상권)정보 API는 국세청·카드사 원천이 명시되어 제3자 권리 범위를 추가 확인해야 하고, 전국 결혼식장·예식장 표준데이터는 현재 서울 제공기관이 없다. 서울 공공예식장 현황은 공공누리 제3유형(변경금지) 파일이며 이번 카드에서 파일을 다운로드하지 않아 필드·가공 적합성을 확정할 수 없다. 묘지·장례식장 원천은 돌잔치 범위 밖이다.

어떤 원천도 돌잔치 가능 여부, 최소보증인원, 가격, 주차, 룸, 식사·케이크·장식 정책, 예약 가능성, 문의 수신 의사, 사진·로고·상세 설명 이용권을 증명하지 않는다. 공공데이터에 존재하거나 `영업/정상`, `운영 Y`로 표시되어도 D-25 승인 전에는 공개 라벨을 구현하지 않는다. 이 보고서의 후보 문구는 `공공데이터 등록` 또는 `정보 확인 중`이며, 최종 라벨·필드별 확인일·재확인 주기는 D-25가 정본이다.

### 권리 해석

- 공공데이터포털 `이용허락범위 제한 없음`: 이 레지스트리에서는 **상업 이용 가능 / 변경 가능**으로 기록한다. 공공누리 번호가 표시된 것은 아니므로 실제 채택 시 해당 문구·확인일·URL을 증거로 저장해야 한다.
- 공공누리 제1유형: 출처표시를 전제로 상업 이용과 변경이 가능하다. 온라인 이용 시 원문 링크를 함께 제공하고 제공기관이 손품해방을 후원·보증하는 것처럼 표시하지 않는다.
- 공공누리 제3유형: 출처표시를 전제로 상업 이용은 가능하지만 변경은 금지된다. 정규화·병합·필드명 변경·부분 발췌가 허용되는지는 별도 판단 전 보류한다.
- 포털 정책상 제3자의 권리가 포함된 경우 정당한 허락을 별도로 받아야 한다. 개별 페이지의 일반적인 `제한 없음` 표기와 제3자 원천 설명이 충돌하면 제3자 확인을 우선한다.

이 문서는 법률 자문이 아니라 제품 도입 전 운영 통제 레지스트리다.

- `15122364`는 묘지, `15122367`은 장례식장 현황이다. 관련 BE-005 가족 생애주기 아키텍처에서 언급되었더라도 **서울 돌잔치 업체 후보 레지스트리에는 목적 적합성이 없다**.
- 장례식장 데이터의 식당·주차·시설 필드가 행사 장소처럼 보이더라도 돌잔치로 전용 추론하지 않는다.
- D-23에서 채택하지 않는다. 장례·추모 영역을 실제 서비스 범위로 추가하려면 사업 범위 승인, 별도 데이터 카드, 개인정보·민감 맥락 검토가 선행되어야 한다.

### 4.10 `15096283` 전국일반음식점표준데이터 — `TERMINATED`

- 확인 근거: 공공데이터포털 표준데이터 페이지는 전국 일반음식점 인허가를 설명하지만, 현재 파일 URL을 종료된 `file.localdata.go.kr/file/general_restaurants/info`로 표시한다. 페이지의 이용허락범위도 공란이다.
- 현재 사용 금지 이유: endpoint가 종료 계열이고 이용허락을 확정할 수 없다. 파일 수정일 2025-11-27과 `매일 갱신·2일 전 기준`이라는 과거 설명을 현재 가동 증거로 보지 않는다.
- 허용 필드: 없음. schema·설명은 전환 대응 참고로만 사용하고 레코드를 호출·다운로드·저장하지 않는다.
- 금지 해석: 표준데이터 상세 페이지가 남아 있다는 사실을 파일 endpoint의 가동, 최신성, 이용허락으로 해석하지 않는다.
- 대체: 같은 업종의 현 조회서비스 `15154916` 상세·명세·권리를 새로 확인해 D-23에서 선택한다. legacy ID와 새 ID는 lineage에 `replaced_by`로 연결하되 동일 ID로 덮어쓰지 않는다.
- 중단 상태: 이미 `TERMINATED`. 공식적으로 endpoint와 license가 모두 새로 확정되지 않는 한 재개하지 않는다.

변경하지 않은 이용 방법이 별도 승인된 경우에만 위 문구의 유형을 제3유형으로 바꿔 사용한다. 승인 전에는 제품에 적재·병합·요약하지 않는다.

## 9. D-23 결정 제안

D-23은 다음을 한 묶음으로 승인해야 한다. 이 보고서 자체는 채택 승인이 아니다.

1. 서울 돌잔치 후보 발견용 canonical source family 선택
   - 우선안: `15154916` + `15154897` + `15155090`
   - 대안: 서울 범위가 필요한 해당 업종만 `OA-16094`/`OA-16091`로 교체
   - 같은 업종에서 전국형·서울형·자치구 mirror를 독립 검증 근거로 중복 채택하지 않음
2. 실제 사용하는 최소 필드와 공개/내부 구분 승인
3. D-25에서 공공데이터 상태와 업체 직접 확인 상태를 분리한 공개 라벨·필드별 확인일·재확인 주기 승인
4. 출처표시 위치·원문 링크·기준일·확인일 계약 승인
5. stale/empty/schema/license/termination 중단 규칙과 철회 lineage 승인
6. `15081808`은 D-24 사업자번호 수집 목적·필수항목·동의·보유기간·접근역할·암호화·파기·이의처리와 secret 통제가 끝난 뒤 상태조회만 조건부 검토. 그 결과의 공개 라벨·확인일은 D-25에서 별도 승인
7. API 활용신청·키 발급·저장·DB 스키마·수집기는 각각 권한이 있는 후속 카드에서 실행

`15012005`, `15155669`, `OA-22377`/`15129614`는 추가 조건이 충족되기 전 D-23 채택안에 넣지 않는다. `15122364`, `15122367`, `15096283`, `LOCALDATA` 직접 endpoint는 채택 금지다.
