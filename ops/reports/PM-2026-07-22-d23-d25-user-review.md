## D-23 공공데이터 추천안

### 추천

행정안전부 전국형 3종을 서울 돌잔치 업체의 **비공개 후보 발견과 행정 상태 변화 신호**에만 사용하는 원천으로 채택한다.

- [15154916 일반음식점](https://www.data.go.kr/data/15154916/openapi.do)
- [15154897 관광식당](https://www.data.go.kr/data/15154897/openapi.do?recommendDataYn=Y)
- [15155090 관광숙박업](https://www.data.go.kr/data/15155090/openapi.do)

2026-07-22 확인 결과 세 원천은 행정안전부 제공, REST JSON/XML, 무료, 일간 갱신, 개발·운영 자동승인, 이용허락 제한 없음으로 표시된다.

### 승인하면 정해지는 것

- 위 3개 dataset ID를 후속 설계의 canonical 원천으로 사용한다.
- 업체명·업종·주소·인허가 상태·관련 일자 등 최소 필드 후보만 설계한다.
- 출처 URL·기준일·확인일·권리조건·중단 이력을 남긴다.
- 서울 mirror는 교체 대안일 뿐 같은 계보를 독립 증거로 중복 계산하지 않는다.

### 승인해도 아직 하면 안 되는 것

- 활용신청·키 발급·API 호출·다운로드·저장·공개
- 공공데이터 업체를 자동으로 공개하거나 검수 업체로 표시
- `영업/정상`을 돌잔치 가능·현재 가격·주차·시설·예약 가능으로 해석
- 대표자·개인 전화·사진·후기·제3자 콘텐츠 수집
- 권리가 불명확한 `15012005` 상가정보 등 보류 원천 사용

`일간 갱신`은 업체를 매일 현장 확인했다는 뜻이 아니다. 일반음식점 대부분은 돌잔치와 무관할 수 있다.

## D-24 개인정보 추천안

### 현재 판정

핵심 원칙은 안전하지만 지금 승인하기에는 문서 보완이 필요하다.

1. QA-013이 참조한 안전조치 고시는 2025-9호다. 현재는 [개인정보의 안전성 확보조치 기준 제2026-9호](https://www.law.go.kr/LSW/admRulInfoP.do?admRulSeq=2100000281400&chrClsCd=010201)가 2026-07-01부터 시행 중이다.
2. 현행 [제5조](https://www.law.go.kr/LSW/admRulSideInfoP.do?admRulSeq=2100000281400&chrClsCd=010201&docCls=jo&joNo=0005&urlMode=admRulScJoRltInfoR)는 개인정보처리시스템 접근권한 부여·변경·말소 이력을 최소 3년 보관하도록 한다. 기존 grant 감사 1년 제안과 구분해야 한다.
3. 현행 [제8조](https://www.law.go.kr/LSW/admRulSideInfoP.do?admRulSeq=2100000281400&chrClsCd=010201&docCls=jo&joNo=0008&urlMode=admRulScJoRltInfoR)의 접속기록은 최소 1년이며 일정 조건에서는 2년이다.
4. OPS-008의 사업자번호 암호화 원문·keyed hash 후보는 최종 QA-013에서 폐기됐다는 점을 명시해야 한다.

### 유지할 핵심 원칙

- 사업자번호는 상태조회 요청 중 서버 메모리에서만 사용하고 응답·오류 직후 즉시 폐기한다.
- DB·Storage·queue·로그·backup에 원문·암호문·hash/token을 저장하지 않는다.
- 사업자 상태, 담당자 관계, 페이지 소유권, 편집 권한, 공개, 문의 활성은 각각 따로 판단한다.
- 초기에는 인물·미성년자 사진과 영수증·계약 같은 후기 증빙을 받지 않는다.
- 관계·사진 권리 같은 다른 원본은 결정 후 30일, 수집 후 최대 90일 제안을 유지하되 법정 고정기간처럼 표현하지 않는다.

### 다음 조치

QA-015에서 위 네 가지를 보완한 뒤 D-24를 다시 제시한다. 그 전 사업자번호·관계서류·사진/후기 증빙의 실제 수집·조회·저장은 0이다.

## D-25 신뢰 표시 추천안

### 추천 묶음

`D25-A1 + D25-B1 + D25-C1 + D25-D1`

- 업체가 낸 정보의 공개 명칭은 `업체 제공 정보`
- 6개 라벨을 업체 전체 점수가 아니라 해당 정보의 의미로 독립 표시
- 경고를 먼저 보여 주고 모바일에는 핵심 2개만 우선 표시
- 가격은 0~90일 현재값, 91~120일 경고, 121일부터 숨김
- 일반 비교정보와 문의는 180일 경계, 담당자 관계·편집 권한은 365일 경계

### 여섯 라벨

1. `공공데이터 등록`
2. `정보 확인 중`
3. `업체 제공 정보`
4. `사업자 확인`
5. `관리자 검수`
6. `최근 정보 확인 필요`

### 꼭 지킬 오인 방지

- 라벨이 여러 개 붙어도 업체 전체가 인증된 것은 아니다.
- `사업자 확인`은 가격·시설·사진·담당자 권한까지 확인했다는 뜻이 아니다.
- `관리자 검수`는 해당 항목의 출처와 표시 조건을 본 것이지 계약 이행을 보증하는 것이 아니다.
- `정보 없음`은 0원·없음·불가를 뜻하지 않는다.
- D-25 승인은 UI·DB·API·배포를 승인하지 않는다.

## 세 결정의 연결

```text
D-23: 후보를 어디서 찾는가
  └─ D-24: 업체 확인 과정의 민감정보를 어떻게 다루는가
      └─ D-25: 고객 화면에서 확인 범위를 어떻게 설명하는가
```

어느 하나도 다른 결정을 대신하지 않는다. 공공데이터에 있다고 개인정보 처리가 허용되는 것도 아니고, 사업자 상태를 확인했다고 모든 업체 정보가 검증되는 것도 아니다.
