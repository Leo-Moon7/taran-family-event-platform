### 신규 업체 등록

계정과 비민감 기본 정보 접수 → 서울/행사 범위 검사 → 기존 업체·지점 중복 확인 → D-24 이후에만 사업자 상태와 담당자 관계 확인 → 등록 수락은 draft로만 생성 → 범위가 명시된 편집 권한 부여 → 업체가 표준 필드를 제출 → 관리자 필드·사진·공개 검수 → 문의 수신 동의와 E2E를 별도 확인한다.

### 기존 페이지 소유권 신청

정확한 업체·지점 선택 → 기존 권한·분쟁·폐업·중복 확인 → 사업자 상태와 요청자의 업체 관계를 각각 확인 → 관리자 승인으로 사용자·업체·역할·허용 범위·만료가 명시된 권한을 발급 → 첫 변경은 기존값을 덮지 않는 pending revision으로 제출 → 승인 필드만 공개한다.

### 업체 직접 입력

업체는 권한 범위 안에서 `이전값 → 제안값`, 기준일, 조건, 근거와 사진 권리를 제출한다. 제출 즉시 공개하지 않는다. 가격·취소·주소·사업자·소유권·문의 정보는 항상 재검수하고, 반려된 필드가 있어도 다른 승인 필드를 자동 취소하지 않는다.

source/rights, 신규 후보, 중복/지점, 주소/폐업, 행사 가능, 사업자 상태, 소유권·권한, 업체 필드, 사용자 제보/견적, 사진 권리, 자체 후기, 최신성/API 실패, 신고·분쟁의 13개 큐를 분리한다. 승인·반려·보완·재제출·권한 회수·이의·rollback에는 판정자, 이유 코드, 전후값, 근거, 시각, 정책 버전을 남긴다.
