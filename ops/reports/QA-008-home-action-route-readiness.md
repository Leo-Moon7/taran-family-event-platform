# QA-008 홈 액션 실제 경로 준비도 점검

- 작업 ID: `QA-008`
- 점검일: 2026-07-22
- 대상 배포: `https://taran-family-event-test.netlify.app/`
- 대상 계획: 승인된 홈페이지 C안의 13개 액션
- 제품 준비도 판정: **BLOCKED**
- 산출물 판정: **COMPLETE — PM 검수 대기**

> `BLOCKED`는 이 점검 산출물의 실패가 아니라, 현재 구현으로 FE-006의 13개 액션을 모두 공개할 수 없다는 제품 준비도 판정이다.

현재 배포 홈은 C안이 아니라 기존 마켓플레이스 홈이다. 배포 홈의 대표 문구는 `가족행사 장소와 업체, 조건을 한눈에 비교하세요.`이며, C안의 정보 보완·커뮤니티·수정 내역·내 준비 현황 액션은 홈에 아직 노출되지 않는다. 이는 FE-006이 선행 검증을 기다리는 현재 상태와 일치한다.

## 2. 판정 기준

- **준비**: 공개 GET 진입, 핸들러, 데이터 위치, 인증 조건, 사용자 결과가 서로 일치한다.
- **부분 준비**: 일부 진입 방식이나 데이터에서만 작동하거나 C안 문구가 실제 결과보다 넓다.
- **차단**: 화면은 있더라도 주요 진입·인증·제출·후속 결과가 끊겨 있어 홈에서 약속하면 안 된다.
- **부재**: 대응하는 공개 경로와 결과 화면이 없다.

## 4. 배포 GET 검증

인앱 브라우저에서 아래 공개 경로를 GET/스크롤로만 확인했다. 폼 제출, 외부 원문 클릭, 로그인, 로컬 상태 변경은 하지 않았다.

- `/`: 기존 홈 제목·검색 GET 폼·6개 행사 카드·계산기·체크리스트·준비백과·입점 링크를 확인했다.
- `/venues?event=kids`: 행사 필터가 적용되고 공개 업체 2건이 렌더링됐다.
- `/venues?event=memorial`, `/venues?event=other`: 필터 선택과 0건 빈 결과가 정상 렌더링됐다.
- `/calculator`, `/checklist`: 입력 컨트롤과 같은 화면 결과 영역이 렌더링됐다.
- `/articles`: `전체 글 28개`와 상세 링크를 확인했다.
- `/community`, `/contact`, `/contribute`, `/provider-join`, `/provider-register`: 각 공개 문서와 폼/목록이 렌더링됐다.
- `/provider?id=review-lifecycle-cc36be385d9f`: `리움하우스` 상세와 ID가 붙은 권한 요청 링크를 확인했다. 후기 원문 데이터·핸들러는 정적 코드로 교차 확인했다.
- `/claim?id=review-lifecycle-cc36be385d9f`: 미로그인 로그인 링크가 원래 claim ID를 return 값으로 보존했다.
- `/account`: 미로그인 상태에서 `/login?return=account.html`로 이동했다.

배포는 Netlify pretty URL을 사용하지만 저장소 HTML은 기존 `.html` URL을 유지한다. 본 점검은 라우팅 변경을 요구하거나 수행하지 않았다.

## 5. FE-006 노출 판단

### 즉시 연결 가능한 범위

- 조건 기반 업체 찾기
- 계산기
- 체크리스트
- 유효한 ID가 있는 업체 상세
- 준비백과
- 커뮤니티 공개 목록

### 조건부 연결 범위

- 행사별 진입: 홈 카드 분류를 확정하고 각 분류의 빈 결과 문구·데이터 준비도를 함께 표시해야 한다.
- 후기 원문: 실제 `externalReviews` 출처가 있는 업체만 링크해야 하며 QA-007·MKT-007 결과를 따라야 한다.
- 업체 권한 요청: 업체 상세의 ID 포함 딥링크만 유지 가능하다. 최상위 독립 CTA는 차단한다.
- 내 준비 현황: 현재 구현에 맞추려면 문구를 `내 계정` 수준으로 낮추거나, 실제 상태 요약 화면을 선행 구현해야 한다.

### 공개하면 안 되는 범위

- 전용 경로·검수 결과·이력이 없는 `정보 수정 제안`
- 대응 화면이 없는 `수정 내역`
- 업체 선택 없이 진입하는 최상위 `업체 권한 요청`
- 인증 선행과 접수 추적이 없는 `신규 입점`

## 7. 변경 요청·승인·회귀

- **CHANGE_REQUEST: 필요.** 정보 수정 제안·공개 수정 내역·claim/registration 상태는 공통 인증/API/DB 계약 영향을 받는다. 본 작업에서는 금지 영역을 수정하지 않았으며 `BIZ-001 → BE-004 → OPS-008 → PM 배정 FE/QA 카드`로 반환한다.
- **APPROVAL_REQUIRED:** 사용자 기여 보상, 공개 변경 이력 범위, 권한/입점 정책(D-15~D-17)은 승인 전 확정하거나 홈에 약속하지 않는다.
- **회귀 영향:** 코드·데이터·라우팅·디자인 토큰을 변경하지 않은 읽기 전용 점검이므로 런타임 회귀 없음.
- **남은 위험:** 실제 폼 제출, 로그인 복귀, RLS, 스토리지 권한, 승인·반려 상태는 카드 제한상 실행하지 않았다. 이는 `QA-003`의 staging/역할 계정 차단이 해소된 뒤 검증해야 한다.
