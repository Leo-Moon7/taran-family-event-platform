## 4. 무료 가치 제안과 금지 약속

업체 가치는 ① 무료 소유권·기본 정정, ② 인원·주차·가격 포함/제외의 표준 입력, ③ 조건이 있는 적합 문의, ④ 프로필 결측·문의 상태의 운영 학습이다. 현재 제품에서 실제 제공되는 범위만 안내한다.

승인용 문안 초안:

> 손품해방은 서울 돌잔치 정보를 같은 조건으로 비교할 수 있도록 준비 중입니다. 업체 담당자는 페이지 소유권 확인과 기본 정보 수정을 무료로 요청할 수 있고, 제출 정보는 관리자 검수 후 `업체 제공 정보`와 확인일로 표시될 수 있습니다. 문의 수신은 별도 동의와 전달 확인 후에만 활성화됩니다. 유료 가입, 상위 노출, 예약 증가를 조건으로 하지 않습니다. 원하지 않으시면 이후 안내를 중단하겠습니다.

금지: `즉시 공개`, `공식 인증`, `추천/대표/최저가`, `상위 노출·독점 문의·매출 보장`, `실시간 가격·예약 가능·예약 확인 후기`, 미확정 가격·수수료, NAVER 후기·평점·4,960개 커버리지.

## 7. 전환·직접 입력·검수

```text
eligible_candidate → contact_approved → delivered/unreachable/declined/do_not_contact
→ claim_started → T4 owner_business_checked → profile_submitted
→ T5 field_reviewed → inquiry_opt_in+E2E → T6 inquiry_ready/paused/stale/revoked
```

T4는 사업자 상태와 담당자 관계를 별도 확인한 상태이며 공개/문의 승인이 아니다. 사업자번호는 자발 제출 뒤 승인된 비공개 경로에서만 확인한다. grant는 업체·scope별로 주고 퇴사·대행 종료·분쟁·오승인 시 회수 이력을 보존한다.

| 직접 입력 | 필수 조건 |
| --- | --- |
| 식별·위치·행사 | 업체/지점·주소/활동지역·공식 채널·돌잔치 제공 조건, canonical/중복 검수 |
| 인원·가격 | 최소/최대/보증, 단위, 식대/대관/세금, 포함·제외, 기준일·유효기간 |
| 주차·시설·정책 | 대수/시간/지원, 접근성, 단독 공간, 외부업체, 취소·변경의 적용 조건 |
| 문의 | responder, 승인 업무 채널, 조건·시간, opt-in과 E2E |
| 사진 | asset별 권리·범위·인물·기간·철회 |

모르는 값은 빈칸/`확인 필요`이며 0·무료·불가로 추정하지 않는다. 초기 90일은 저위험 필드도 전부 관리자 검수하며 가격·취소·주소·사업자·소유권·문의는 항상 재검수한다.

큐는 `후보`, `소유권/사업자`, `업체 필드`, `사진`, `문의`, `후기`, `오류·폐업`, `신고·분쟁·철회`로 분리한다. 결정마다 전후값·근거·사유·정책 버전·검수자·시각·되돌림을 남긴다. 제출자와 승인자를 분리하고 초기 10개 T4/T5는 별도 승인권자가 100% 재검토한다.

사유 코드: `duplicate_or_branch_unresolved`, `identity_or_business_mismatch`, `relationship_insufficient`, `event_fit_unsubstantiated`, `scope_or_date_missing`, `price_conditions_incomplete`, `photo_rights_unknown`, `person_release_missing`, `personal_information_exposed`, `misleading_claim`, `inquiry_route_unverified`, `stale_or_conflicting`.

## 8. 사진 제공 동의

사진은 프로필 필수가 아니며 권리 근거가 없으면 placeholder를 쓴다. 공개 asset마다 provider/asset ID, 제출자와 허락 권한, 직접 촬영·보유·라이선스 등 권리 근거, 상세/검색/콘텐츠/외부광고별 범위, 편집·출처표시, 시작·만료·철회, 인물·미성년자 동의, 검수·분쟁 이력이 필요하다.

업체 홈페이지·SNS 사진을 발견했다는 이유로 복사하지 않는다. 업체의 게시 권한과 손품해방 사용 권한도 분리한다. 철회 시 공개 파생본을 차단하고 placeholder로 전환한다. 이는 체크리스트이지 법률 동의서가 아니며 실제 수집·보유·공개는 개인정보·권리 정책 승인 후다.

| 실제 경험 자기보고 | `이용자 후기` | 로그인, 행사 시점·조건·관계·광고 여부, pending 검수 |
| 비공개 증빙 확인 | `이용 증빙 확인` | 최소수집·보유·파기·이의 정책 승인 후만 파일럿 |
| 플랫폼/외부 예약 대조 | `예약 확인 후기` | 현재 비활성·0건; 예약·증빙·분쟁 정책과 제품 승인 후 검토 |

`자체 후기`는 플랫폼이 긍정 문구를 쓰는 것이 아니라 손품해방에서 직접 접수·검수한 1차 후기다. 개인정보·광고·이해관계 미표시·욕설·무근거 비방·조작·중복을 검수하고 신고→업체 소명→임시 숨김→이의→복구/삭제 이력을 둔다. 후기·평점은 업체 공개 자격·무료 수정권과 분리한다.

후기 요청 이메일/SMS도 외부 연락이므로 동의·채널·횟수·문안 승인 전 실행하지 않는다. 인센티브는 비용·표시 승인 후이며 긍정 후기·특정 평점을 조건으로 하지 않는다. 모집단·기간·중복·광고·최소 표본·정렬 정책 전에는 평균 평점, `후기 많은/평점 좋은/추천`을 쓰지 않는다.

## 11. KPI·운영 용량·중단

퍼널: `eligible → approved_contact → delivered → interested → claim → T4 → submitted → T5 → T6`. 자발 신청과 아웃바운드는 분리하고 무응답·반려를 분모에서 빼지 않는다. 품질은 출처·제공자·최근 확인일 100%, 미해결 중복/폐업·무권한 편집·권리 불명 사진·예약 확인 오표시·승인 밖 연락·수신거부 재접촉 각 0건이다.

유효 문의는 테스트·스팸·중복이 아니고 동의·연락 수단·행사·희망 시점·인원·지역·필수 조건이 있으며 선택한 T6에만 전달된 건이다. 문의 본문·연락처·사업자번호·증빙은 분석 metadata에 넣지 않는다. 주간판은 각 단계 신규/누적, 큐 유입·처리·잔량·최고 체류·재작업, 문의 전달·열람·응답·실패, 콘텐츠/후기 상태, 안전 사건과 다음 배치를 같은 분모로 기록한다.

역할 슬롯은 acquisition operator, provider reviewer, rights/content reviewer, approver/incident owner다. 실제 인력·운영시간은 미확정이다. 큐가 향후 5영업일 관측 용량을 넘거나 고위험 건이 남으면 신규 연락·후기 모집을 중단하며 외부에 당일/24시간 처리를 약속하지 않는다.

즉시 중단: NAVER 재사용, 승인 밖 연락, 수신거부 재접촉, 첫 10건 claim 0, 두 배치 T4/T5 0, 무권한 편집·PII/증빙 노출, 사진 권리 위반, 문의 실패 성공 오표시·선택 외 전달, 예약 확인 오표시, 광고 미표시, 고위험 신고 담당 부재.

## 12. 전국 확장·수익 보류 게이트

서울에서 ① 출처·주체·최근일 100%, ② 중복·폐업·권리·PII·광고·예약 후기 사고 0, ③ T6 20곳과 분포, ④ 유효 전달 30건·전달 95%·응답 50% 제안, ⑤ 내부 큐 목표 90%와 잔여 용량, ⑥ RLS·알림·실패·철회·신고 E2E를 모두 통과해야 한다.

다음 `지역×행사` 한 코호트는 반복 수요, 승인된 독립 후보, 행사 고유 비교 항목, 서울을 해치지 않는 운영 여력, 품질·보안 PASS, 업체 가치 학습, 고유 콘텐츠·T5/T6 SEO 준비를 점수화한 뒤 D-08과 사용자 승인으로 정한다. 전국 일괄 영업·대량 빈 SEO 페이지는 금지한다.

90일 수익 활동은 무료 소유권·프로필·문의 가치, 운영 원가, 업체가 인정한 유효 문의, 지불의향을 학습하는 데 그친다. Premium·구독·스폰서·리드 과금·제휴·수수료는 미확정이고 공개/영업하지 않는다. 예약·결제 수수료는 재고·계약·환불·세금·정산·분쟁·법무 후속 단계다.
