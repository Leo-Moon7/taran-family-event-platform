# SONPUM HAEBANG 운영센터 기획 수정안

## 1. 운영센터의 목적

SONPUM HAEBANG 운영센터는 개발자가 코드를 만지는 화면이 아니라, 운영자가 매일 확인하고 바로 수정하는 업무 화면으로 설계합니다.

가장 중요한 기준은 다음 세 가지입니다.

- 오늘 처리해야 할 견적·업체·콘텐츠 상태가 먼저 보여야 합니다.
- 배너, 게시글, 업체 정보는 코드를 몰라도 입력칸만 채우면 수정할 수 있어야 합니다.
- 실제 온라인 운영에서는 Supabase 로그인과 DB 저장을 연결해 관리자만 수정할 수 있어야 합니다.

## 2. 메뉴 구조

첨부 `11.html`의 방향을 참고해 운영 메뉴를 실무 순서로 재정렬합니다.

1. 오늘 할 일
   - 견적 공유 검토
   - 업체 보강 대기
   - 진행 중 배너
   - 최근 작업

2. 견적 관리
   - 고객이 공유한 견적서, 사진, 업체 정보를 검토합니다.
   - 가격, 수용 인원, 주차, 룸, 포함 항목처럼 업체 조건표에 반영할 정보를 추출합니다.
   - 개인정보가 포함된 원본은 검토 후 보관하지 않는 기준으로 운영합니다.

3. 업체 관리
   - 공개 업체와 검토 대기 업체를 분리합니다.
   - 주소, 행사 유형, 참고 후기 또는 공식 채널, 가격·수용 인원 중 최소 일부가 확인된 업체부터 공개합니다.
   - 업체 담당자가 직접 수정 요청할 수 있는 흐름으로 확장합니다.

4. 게시판 관리
   - 공지사항, 준비백과, FAQ, 커뮤니티 글을 한곳에서 관리합니다.
   - AI가 초안을 작성하더라도 최종 공개 전 운영자가 제목, 본문, 이미지, 공개 상태를 확인합니다.

5. 배너 관리
   - 메인 배너, 업체 목록 배너, 준비백과 배너, 계산기 배너를 운영자가 직접 교체합니다.
   - 시작일, 종료일, 공개/숨김 상태를 관리합니다.

6. 회원 관리
   - 고객 계정, 포인트, 최근 활동, 제보 내역을 확인합니다.
   - 견적 검토 업무와 회원 상태 관리를 분리해 헷갈리지 않게 합니다.

7. 통계
   - 방문수, 업체 상세 클릭, 견적 공유, 준비백과 조회를 확인합니다.
   - 추후 검색어, 지역, 행사 유형별 전환도 추가합니다.

## 3. 비개발자용 입력 원칙

운영자가 직접 다루는 화면에는 다음 표현을 쓰지 않습니다.

- JSON
- 슬러그
- DB 스키마
- content-overrides.js
- import/export

대신 다음처럼 바꿉니다.

- 게시글 주소(URL)
- 후기 제목 / 링크 주소 / 작성일
- 온라인 저장
- 관리자 백업
- 공개 / 임시저장 / 숨김

## 4. 업체 공개 기준

업체는 무조건 많이 공개하기보다, 고객이 비교에 사용할 수 있는 최소 정보가 있는 곳부터 공개합니다.

공개 권장 기준:

- 주소 또는 지역이 확인됨
- 가능한 행사 유형이 확인됨
- 참고 후기 또는 공식 채널이 있음
- 가격, 수용 인원, 주차, 룸, 포함 항목 중 1개 이상 확인됨

정보가 부족한 업체는 “검토 대기”로 남기고, 관리자 또는 업체 담당자가 보강한 뒤 공개합니다.

## 5. Supabase 연결 방향

운영 버전은 아래 구조로 갑니다.

- Supabase Auth: 관리자 로그인
- Supabase DB: 업체, 게시글, 배너, 고객, 견적 제보 저장
- Supabase Storage: 업체 사진, 배너 이미지, 견적서 첨부 파일 저장
- 관리자 권한: 승인된 관리자만 수정 가능
- 공개 페이지: 공개 상태의 데이터만 노출

현재 정적 프로토타입은 이 구조를 염두에 둔 화면 설계 단계입니다.
