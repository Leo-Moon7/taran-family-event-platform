# 손품해방 프로젝트 핵심 규칙

이 저장소는 기존 손품해방 가족행사 플랫폼을 지속 개선한다. 사이트를 새로 작성하지 않고 현재 URL, 데이터 계약, 디자인 토큰, Supabase 권한 구조와 완료 산출물을 보존한다.

## 기준 우선순위

1. 사용자가 최근 명시적으로 확정한 내용
2. `docs/99_의사결정기록.md`
3. `AGENTS.md`
4. `docs/12_통합실행계획.md`
5. 승인된 활성 작업 카드
6. 분야별 기존 문서
7. 과거 보고서·초안

충돌은 임의 해석하지 않고 `ops/RISKS.md` 또는 `ops/APPROVALS.md`에 기록한다.

## 총괄 PM 운영

- 루트 대화가 총괄 PM이며 기본 운영 방식은 프로젝트 custom agent를 이용한 서브에이전트 오케스트레이션이다.
- 전체 실행 루프, 에이전트 매핑, 검수·수정·중단 규칙은 `ops/PM_ORCHESTRATION.md`를 따른다.
- 세부 파일 소유권은 `ops/FILE_OWNERSHIP.md`, 의존성과 병렬 가능 여부는 `ops/DEPENDENCIES.md`를 따른다.
- 모든 작업은 `ops/TASK_SPECS.md` 카드가 있어야 하며 실제 실행 중에는 `ops/ACTIVE_WORK.md`에 등록한다.
- 동시에 실행하는 전문 작업은 최대 3개다. 파일, API, DB, 라우팅, 디자인 토큰, 환경변수, 패키지·공통 설정이 겹치면 병렬 실행하지 않는다.
- CHG-A~C 등 기존 미할당 변경은 소유권이 정해질 때까지 수정·되돌림·병합하지 않는다.
- 범위 밖 문제는 직접 고치지 않고 기존 백로그와 중복 검사 후 후보로 보고한다.

## 승인·안전 게이트

운영 배포, 최종 병합, 운영 DB·개인정보 변경·삭제, 결제, 비용, 외부 게시·업체 연락, 법적 문서, 파괴적 마이그레이션, 브랜드·서비스 범위·가격·수수료·주요 기능 삭제는 사용자 승인 전 실행하지 않는다.

공통 파일·API·DB·라우팅·환경변수·패키지 변경이 필요하면 전문 에이전트는 작업을 중단하고 총괄 PM에게 변경 요청을 반환한다.

## 완료 보고

작업 ID, 결과, 수정·추가·삭제 파일, 실행 검증, 완료 조건, 재현 근거, 영향, 남은 문제, 변경 요청, 승인 필요, 병합 권고를 총괄 PM에게 보고한다. 총괄 PM의 검수에서 `PASS`를 받은 작업만 `DONE`이다.
