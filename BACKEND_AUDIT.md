# 백엔드 연결 감사

## 현재 상태

- 공개 페이지의 기본 데이터: 저장소 JavaScript 파일
- 브라우저 임시 저장: localStorage
- 관리자 온라인 저장 후보: Supabase REST
- 레거시 인증: 별도 `/api/...` 호출 가정

서로 다른 방식을 동시에 운영하지 않도록 최종 구조는 Supabase 중심으로 통일하는 것이 현재 규모와 비개발자 운영에 적합합니다.

## 공개하면 안 되는 미완성 기능

- 실제 관리자 로그인과 권한 확인
- 온라인 업체·글·배너 수정
- 견적 문의 서버 접수
- 기프티콘 포인트 정산
- 비공개 파일 업로드
- 실제 방문·매출 통계

연결 전에는 위 기능을 성공한 것처럼 표시하지 않으며 관리자 경로는 호스팅 접근 제한을 적용합니다.

## 권장 구성

- 인증: Supabase Auth
- DB: PostgreSQL
- 파일: 비공개 Supabase Storage
- 권한: RLS + `taran_admin_profiles`
- 민감 작업: Edge Functions

## 보안 원칙

- 서비스 역할 키와 DB 비밀번호를 프론트엔드에 포함하지 않습니다.
- CMS 문구는 사전 정의된 `contentSlotId`와 일반 텍스트만 허용합니다.
- 링크는 `http`, `https`, `tel`, `mailto`만 허용하고 `javascript:`를 차단합니다.
- 업체명·주소·후기는 `textContent`로 출력합니다.
- 업로드는 확장자, MIME, 크기, 권한, 보유기간을 서버에서 검사합니다.
