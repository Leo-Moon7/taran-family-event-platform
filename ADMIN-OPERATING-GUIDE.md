# 따란 관리자 운영 안내

관리자 주소는 `/admin/`입니다. Supabase가 연결된 공개 주소에서는 관리자 권한이 있는 로그인 계정만 접근할 수 있습니다.

## 매일 확인할 순서

1. `오늘 할 일`: 새 견적, 정보 공유, 공개 대기 항목 확인
2. `견적 관리`: 고객 견적 문의 상태 변경, 정보 공유 승인·반려
3. `업체 관리`: 업체 정보 수정, 소유권 요청 승인·반려, 공개 대기 후기 검수
4. `게시판 관리`: 준비백과 글 작성·수정·공개, 커뮤니티 공개 대기 글 검수
5. `배너 관리`: 게시 위치와 기간을 확인한 뒤 공개
6. `회원 관리`: 회원 상태와 탈퇴 요청 확인
7. `통계`: 실제 페이지 이용 기록과 견적 흐름 확인

## 업체를 공개하는 기준

다음 필수 기준이 모두 확인된 업체만 `공개`로 전환합니다.

- 업체명과 도로명 주소 또는 정확한 활동 지역
- 공식 채널 또는 연락 가능한 대표 정보
- 업체 유형과 가능한 행사 유형
- 정보 확인일

아래 조건은 확인된 항목만 입력하고, 모르면 빈칸으로 둡니다.

- 가격·식대·대관료
- 최소·최대·보증 인원
- 주차 가능 대수와 무료 시간
- 외부 음식·외부 업체 이용 조건
- 취소·환불 규정

후기에서 합리적으로 유추한 내용은 내부 검수 메모로만 관리하고, 고객 화면에는 사실로 확인된 값만 공개합니다.

## 업체 담당자 권한 승인

1. 업체 관리의 `소유권 요청`에서 업체명과 담당자 정보를 확인합니다.
2. 비공개 사업자 확인 자료를 열어 상호와 사업자등록번호를 대조합니다.
3. 요청 계정이 실제 담당자임을 확인한 경우에만 `승인`합니다.
4. 승인 즉시 해당 계정의 내 정보 화면에 `내 업체 관리`가 나타납니다.
5. 업체 담당자는 승인받은 업체만 수정할 수 있으며 공개 상태 자체는 바꿀 수 없습니다.
6. 잘못 승인한 경우 최고관리자가 업체의 `owner_user_id`를 비우고 요청 상태를 반려로 변경합니다.

## 커뮤니티 검수

- 새 게시글은 기본적으로 공개 대기 상태로 접수됩니다.
- 개인정보, 광고성 링크, 욕설, 업체를 특정한 근거 없는 비방이 없는지 확인합니다.
- 공개할 글만 승인하고, 문제가 있는 글은 숨김 처리합니다.
- 댓글 신고·차단 기능을 추가하기 전까지 운영자가 정기적으로 댓글을 확인합니다.

## 정보 공유와 포인트

- 견적서·사진은 개인정보가 가려진 자료만 승인합니다.
- 승인 시 시스템이 예상 포인트 범위에서 한 번만 지급합니다.
- 반려 사유는 개인정보 노출, 업체 식별 불가, 중복 제출, 조작 의심 등으로 기록합니다.
- 기프티콘 교환은 포인트·재고를 동시에 확인한 뒤 접수되며, 발송 완료 후 상태를 `sent`로 바꿉니다.

## 준비백과 글 작성

- 제목은 독자가 검색할 질문에 바로 답하도록 씁니다.
- 본문은 일반 텍스트로 입력하며 HTML 코드를 붙여넣지 않습니다.
- 가격·규정처럼 바뀔 수 있는 정보에는 기준일과 확인 방법을 적습니다.
- 업체 홍보나 제휴 링크는 광고·제휴임을 표시합니다.

## 회원과 개인정보

- 회원 연락처를 내려받아 별도 명단으로 보관하지 않습니다.
- 견적 문의가 완료되면 불필요한 연락처 사본을 만들지 않습니다.
- 탈퇴 요청은 본인 계정의 저장 목록·준비 상태가 먼저 삭제됩니다. Auth 계정과 법정 보관 대상이 아닌 잔여 자료는 운영자가 확인 후 삭제합니다.
- 관리자 화면을 공용 PC에서 사용한 뒤 반드시 로그아웃합니다.

## 문제가 생겼을 때

- 저장 버튼이 보이지 않음: Supabase 설정과 관리자 권한 확인
- 로그인 후 관리자 접근 불가: `taran_admin_profiles` 역할 확인
- 공개 사이트에 반영 안 됨: 데이터의 `status`가 `published`인지 확인
- 파일 업로드 실패: JPG·PNG·WEBP·PDF, 10MB 이하인지 확인
- SQL 오류: 실행한 문장과 오류 메시지를 보관하고 임의로 정책을 삭제하지 않습니다.
