# 따란 오픈 전 전체 점검표

작성일: 2026-07-07  
범위: `taran-haebang-upgraded` 로컬 시제품 기준

이 문서는 “지금 사이트를 바로 외부에 공개해도 되는가?”를 판단하기 위한 실행용 점검표입니다.  
현재 결론은 **화면 시제품은 꽤 잘 잡혔지만, 실제 오픈 전에는 P0 항목을 먼저 처리해야 함**입니다.

---

## 1. 현재 시제품 상태 요약

### 확인 완료

- 주요 화면 13개가 로컬 서버에서 정상 응답
- 주요 JavaScript 파일 문법 검사 통과
- 고객 화면과 운영자 화면의 큰 흐름 연결 완료
- 크롤링 데이터는 바로 공개하지 않고 검수 대기 → 관리자 승인 → 공개 흐름으로 설계됨
- 업체 권한 신청 → 관리자 검수 → 업체 대시보드 흐름 있음
- 업체 전체보기는 전체 카테고리 디렉토리로 확장됨

### 현재 주요 화면

| 구분 | 화면 | 상태 | 비고 |
| --- | --- | --- | --- |
| 고객 | 메인 | 완료도 높음 | 최종 문구/모바일 QA 필요 |
| 고객 | 업체 전체보기 | 완료도 높음 | 전체 카테고리 확장 완료 |
| 고객 | 장소 상세 | 완료도 높음 | 실제 데이터 전환 전 문구 정리 필요 |
| 고객 | 일반 업체 상세 | 기본 완료 | 고객용 상세 구체화는 현재 보류 |
| 고객 | 계산기 | 기본 완료 | 입력 항목 최종 검토 필요 |
| 고객 | 체크리스트 | 기본 완료 | 커머스/제휴 링크는 추후 |
| 고객 | 로그인/내 정보 | 시제품 | 실제 개인정보 수집 전 백엔드 필요 |
| 운영자 | 관리자 검수 센터 | 기본 완료 | 실제 관리자 인증 필요 |
| 업체 | 정보 수정 권한 신청 | 기본 완료 | 실제 사업자 문서 업로드는 백엔드 필요 |
| 업체 | 업체 대시보드 | 기본 완료 | 승인된 업체만 접근하도록 서버 권한 필요 |
| 영업 | 입점 안내 | 기본 완료 | 요금제 문구 최종 확정 필요 |

## 3. P0: 외부 공개 전 필수 처리

### 3-1. 시제품 문구와 실제 서비스 문구 분리

- [ ] `시제품`, `예시`, `검증용` 문구를 유지할 화면과 제거할 화면 구분
- [ ] 고객에게 공개할 화면에서는 “예시 업체”와 “실제 검수 업체”를 명확히 구분
- [ ] 실제 업체가 등록되기 전에는 가격/상호가 실제처럼 오해되지 않게 표시
- [ ] 관리자/업체 화면은 공개 검색엔진에 노출되지 않게 처리

판단: 지금은 시제품 안내가 곳곳에 들어가 있어 안전하지만, 실제 공개 전에는 화면별 문구 정리가 필요합니다.

---

### 3-2. 개인정보 수집 구조 정리

현재 로그인/권한 신청 화면에는 이름, 이메일, 전화번호, 사업자등록증 업로드 흐름이 있습니다.  
실제 오픈에서는 로컬 저장소나 정적 HTML만으로 개인정보를 수집하면 안 됩니다.

- [ ] 개인정보처리방침 작성
- [ ] 수집 항목 명시: 이름, 이메일, 전화번호, 업체 담당자 정보 등
- [ ] 수집 목적 명시: 계정 관리, 관심 업체 저장, 권한 신청 검수 등
- [ ] 보유 기간 및 삭제 요청 방법 명시
- [ ] 전화번호/사업자등록증 등 민감한 운영 자료는 실제 서버 저장 구조 확정 후 수집
- [ ] 로그인/회원가입 동의 체크박스 문구 정리
- [ ] 사업자등록증 파일 업로드는 실제 보안 저장소가 준비되기 전까지 비활성화 또는 안내용으로만 유지

참고 공식 자료:

- 개인정보보호위원회/개인정보 포털: [2026 개인정보 처리방침 작성지침](https://www.privacy.go.kr/front/bbs/bbsView.do?bbsNo=BBSMSTR_000000000049&bbscttNo=20885)
- 개인정보보호위원회 안내서 목록: [개인정보 처리방침 작성지침 2026.4 개정](https://www.pipc.go.kr/np/cop/bbs/selectBoardList.do?bbsId=BS217&mCode=D010030000)

### 3-3. 실제 로그인/저장 방식 결정

현재 로그인은 시제품용입니다. 실제 오픈에서는 고객이 저장한 체크리스트, 관심 업체, 업체 권한 요청을 서버에 저장해야 합니다.

- [ ] 로그인 방식 결정: 이메일 로그인 먼저 / 카카오·네이버 나중
- [ ] 고객 데이터 저장 위치 결정
- [ ] 업체 담당자 권한 저장 구조 결정
- [ ] 관리자 승인 상태 저장 구조 결정
- [ ] 비밀번호 또는 인증 방식 보안 검토
- [ ] 로컬스토리지 기반 저장은 데모 용도로만 유지

추천: 초기에는 카카오/네이버 로그인보다 **이메일 기반 간단 로그인 + 서버 DB**가 더 현실적입니다.

---

### 3-4. 관리자 화면 접근 제한

현재 `admin-review.html`은 URL을 알면 접근 가능한 시제품 화면입니다.

- [ ] 외부 공개 시 관리자 화면은 반드시 로그인/권한으로 보호
- [ ] `admin-review.html`은 검색엔진 `noindex` 유지
- [ ] 관리자 승인/보류 버튼은 실제 서버 권한 검증 후 동작
- [ ] 검수 이력 로그 저장
- [ ] 누가 언제 어떤 업체를 발행했는지 기록

---

### 3-5. 크롤링 데이터 공개 원칙 확정

현재 방향은 좋습니다. 크롤링 → 승인 대기 → 관리자 검수 → 공개 흐름을 유지해야 합니다.

- [ ] 크롤링한 업체는 기본값 `pending`
- [ ] 공개 목록에는 `published` 업체만 노출
- [ ] 수집 출처 URL 저장
- [ ] 수집일과 최종 확인일 분리
- [ ] 전화/공식 채널 확인 여부 저장
- [ ] 가격 기준일 저장
- [ ] 숨은 비용 확인 항목 저장
- [ ] 폐업/이전/가격 변경 가능성 표시
- [ ] 크롤링 대상 사이트의 약관/robots 정책 확인
- [ ] 블로그/카페 글의 문장·사진을 그대로 복사하지 않기

---

### 3-6. 사업자 정보와 약관 정리

사이트에서 회원가입, 업체 문의, 광고 상품, 향후 결제/중개/커머스를 다룰 예정이므로 기본 사업자 정보와 약관 정리가 필요합니다.

- [ ] 푸터에 사업자명, 대표자, 사업자등록번호, 연락처, 이메일 표시 여부 결정
- [ ] 이용약관 작성
- [ ] 개인정보처리방침 작성
- [ ] 업체 입점/광고 상품 약관 또는 안내문 작성
- [ ] 통신판매업 신고 필요 여부 확인
- [ ] 결제/커머스/중개 수수료를 붙이기 전 법무·세무 확인

참고 공식 자료:

- 공정거래위원회 통신판매사업자 조회: [통신판매사업자조회](https://www.ftc.go.kr/www/selectBizCommList.do?key=254)
- 공정거래위원회 등록현황 안내: [통신판매업 등록현황](https://www.ftc.go.kr/www/bizContents.do?key=253)

### 3-7. 도메인/호스팅 전 기술 점검

- [ ] `file://` 기준이 아니라 `https://도메인` 기준으로 모든 화면 확인
- [ ] 모든 내부 링크 확인
- [ ] 이미지 경로 확인
- [ ] CSS/JS 캐시 버전 정리
- [ ] 모바일/PC 주요 해상도 확인
- [ ] 404 페이지 준비
- [ ] robots.txt 준비
- [ ] sitemap.xml 준비
- [ ] Google Search Console / Naver Search Advisor 등록 준비
- [ ] 관리자/시제품 화면은 sitemap에서 제외

현재 발견한 보완점:

- 일부 화면은 `upgrade.css`, `data.js`를 버전 없이 불러옵니다.
- 브라우저 캐시 때문에 수정이 늦게 반영되는 일이 있었습니다.
- 오픈 전에는 CSS/JS 버전 정책을 한 번에 정리하는 것이 좋습니다.

## 6. 권장 실행 순서

### 1순위: 외부 공개 가능 상태 만들기

1. 시제품 문구 정리
2. 개인정보처리방침/이용약관 초안 작성
3. 관리자 화면 접근 제한 방식 결정
4. CSS/JS 캐시 버전 정리
5. 고객 화면 전체 QA

### 2순위: 실제 업체 DB 준비

1. 업체 DB 시트 만들기
2. 검수 상태값 확정
3. 10개 실제 후보 업체를 수동 입력
4. 직접 검수 후 공개 가능 데이터만 선별
5. 공개 페이지에 실제 업체 10개 반영

### 3순위: 크롤링 준비

1. 수집 대상 유형 정리
2. 수집 가능한 출처 정리
3. 크롤링 결과를 `pending`으로 저장
4. 관리자 검수 후 `published` 전환
5. 공개 페이지 자동 반영

### 4순위: 도메인/호스팅

1. 도메인 확정
2. 정적 호스팅 또는 워드프레스/백엔드 방식 결정
3. HTTPS 적용
4. sitemap/robots 준비
5. 검색엔진 등록

## 7. 지금 바로 다음에 할 작업 추천

다음 작업은 **“오픈 전 P0 점검 중 1번: 시제품 문구 정리 + 공개/비공개 화면 구분”**입니다.

이유:

- 현재 사이트는 기능이 많아졌지만, 고객에게 보여도 되는 화면과 운영자용 화면이 섞여 있습니다.
- 실제 오픈 전에는 고객 화면에서 불필요한 “시제품/예시” 문구를 정리하고, 반대로 실제 데이터가 아닌 곳은 오해 방지 문구를 남겨야 합니다.
- 이 작업을 해야 도메인/호스팅으로 넘어가도 리스크가 줄어듭니다.
