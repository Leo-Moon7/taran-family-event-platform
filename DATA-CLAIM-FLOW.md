# 손품해방 데이터 검수·업체 권한 운영안

## 1. 크롤링 데이터 발행 원칙

1. 크롤러가 수집한 레코드는 `pending` 상태로만 저장합니다.
2. 주소, 공식 출처, 가격 기준일, 연락 가능 여부를 관리자가 확인합니다.
3. 검수에 통과한 레코드만 `published`로 변경하여 공개 API에 포함합니다.
4. 부실하거나 폐업 가능성이 있는 레코드는 `rejected`로 보류하고 공개하지 않습니다.

워드프레스 사용 시 REST API로 신규 업체를 보낼 때 `status: pending`을 기본값으로 사용합니다. 공개 화면 쿼리는 `publish` 상태만 조회해야 합니다.

## 2. 업체 소유권 요청

1. 공개 업체 페이지에서 담당자가 수정 권한을 요청합니다.
2. 사업자등록증, 상호·대표자, 업무 연락처, 업체 연관성을 관리자가 확인합니다.
3. 승인된 계정에 해당 업체 ID 한 개의 편집 권한만 부여합니다.
4. 업체 수정 내용에는 수정 주체와 최종 갱신일을 기록합니다.

사업자등록증은 공개 저장소에 두지 않습니다. 비공개 저장소에서 관리자만 열람하고, 검수 완료 후 정책에 따라 파기해야 합니다.

## 3. 역할과 권한

- `admin`: 크롤링 검수, 발행·보류, 업체 소유권 승인, 모든 업체 수정
- `vendor`: 승인받은 본인 업체만 수정
- `member`: 관심 업체 저장과 체크리스트 이용
- `guest`: 공개 업체와 가이드 열람

프론트엔드에서 버튼만 숨기는 방식으로 권한을 제어하면 안 됩니다. 실제 수정 API가 로그인 계정의 역할과 업체 ID를 서버에서 다시 검증해야 합니다.

## 4. 광고 상품

- Basic 무료: 기본 정보·연락처·가격 수정, 일반 광고 영역 유지
- Premium 월 39,000원 예정: 검색 상단 스폰서, 미디어 확대, 경쟁 광고 제거, 직접 상담 버튼

소유권 승인과 정보 수정은 무료로 제공하고, Premium은 별도 선택 상품으로 안내합니다. 무료 권한 제공을 광고 구매 조건으로 묶지 않습니다.

## 5. 현재 시제품 범위

- 브라우저 `localStorage`로 승인 상태와 업체 수정 내용을 시뮬레이션합니다.
- 사업자 문서는 선택 여부만 확인하며 업로드하거나 저장하지 않습니다.
- 실제 로그인, 서버 DB, 관리자 보안, 결제, 정산 기능은 연결되어 있지 않습니다.
